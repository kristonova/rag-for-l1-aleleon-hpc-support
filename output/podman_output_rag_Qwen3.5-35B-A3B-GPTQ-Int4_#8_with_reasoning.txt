[Q8/69] Apa itu PKSPIAS dalam pendaftaran akun ALELEON?

[Q9/69] Jika saya pakai aplikasi SFTP seperti FileZilla, apakah ada limit ukuran file yang bisa diupload?
------------------------------------------------------------
Ya, terdapat batasan ukuran file saat menggunakan aplikasi SFTP seperti FileZilla untuk transfer data ke direktori HOME. Berdasarkan dokumen referensi, limit upload adalah 100MB per file. Jika ukuran file Anda melebihi 100MB, Anda disarankan untuk tidak menggunakan metode tersebut atau memastikan file dipecah sebelum diunggah, karena dokumen secara eksplisit menyatakan bahwa bila lebih dari batas tersebut harus menggunakan aplikasi SFTP (yang mengindikasikan bahwa limit ini berlaku pada konteks transfer via web service, namun instruksi spesifik untuk SFTP dalam konteks HOME menyebutkan "Bila lebih gunakan aplikasi SFTP" yang berarti jika lewat Web Service ada limit, tapi lewat SFTP sendiri tidak disebutkan ada limit).

Namun, perlu diperhatikan bahwa dokumen membedakan antara transfer via **EFIRO Web Service** dan **Aplikasi SFTP**:
1. Untuk transfer via **EFIRO Web Service**, limit upload adalah **100MB per file**.
2. Untuk transfer via **Aplikasi SFTP** (seperti FileZilla) ke direktori HOME, dokumen menyatakan **"Tidak ada limit ukuran upload / download"** pada bagian yang merujuk pada opsi SFTP secara umum, meskipun ada catatan kecil di atasnya yang mengatakan "Bila lebih gunakan aplikasi SFTP".

Jadi, jika Anda menggunakan aplikasi SFTP (FileZilla) untuk upload ke direktori HOME, **tidak ada limit ukuran file** yang diterapkan berdasarkan informasi yang tersedia di dokumen.

[Q12/69] Bagaimana cara membatalkan/menghentikan job yang berstatus PENDING di terminal?
------------------------------------------------------------
Untuk membatalkan atau menghentikan job yang berstatus PENDING di terminal, Anda dapat menggunakan perintah `scancel` diikuti dengan ID job tersebut. Berikut adalah langkah-langkahnya:

1. Lihat status dan ID job Anda dengan menjalankan perintah `$ squeue -ul $USER`.
2. Identifikasi ID job yang ingin dibatalkan dari kolom output (misalnya job ID 231).
3. Jalankan perintah pembatalan dengan format `$ scancel [job-ID]`, contoh spesifiknya adalah `$ scancel 231`.

Perintah ini akan mengubah status job menjadi CA (CANCELED) yang berarti job dibatalkan oleh user.

[Q21/69] Saya ingin buka sesi interaktif JupyterLab menggunakan GPU. Apa bedanya partisi torti dan tilla, dan mana yang harus saya pilih?
------------------------------------------------------------
Partisi **torti** adalah *interactive node* yang hanya menyediakan sumber daya CPU, sedangkan partisi **tilla** adalah *interactive node* yang dilengkapi dengan GPU NVIDIA RTX 3090 24GB.

Untuk membuka sesi interaktif JupyterLab menggunakan GPU, Anda harus memilih partisi **tilla**.

Berikut adalah perbedaan spesifikasi detail antara kedua partisi tersebut berdasarkan dokumen:

1.  **Partisi torti**:
    *   Berfungsi sebagai *interactive node* CPU.
    *   Menggunakan CPU AMD Threadripper 3990X (32 v-core, 2.9Ghz).
    *   Memiliki RAM efektif 55GB per node.
    *   Tidak memiliki GPU.

2.  **Partisi tilla**:
    *   Berfungsi sebagai *interactive node* GPU.
    *   Menggunakan CPU AMD Threadripper 3990X (32 v-core, 2.9Ghz).
    *   Memiliki RAM efektif 55GB per node.
    *   Dilengkapi dengan 1x GPU NVIDIA RTX 3090 24GB dengan arsitektur NVIDIA Ampere.
    *   Catatan penting: Partisi ini tidak mendukung CUDA versi 11 kebawah.

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Berikut adalah urutan langkah untuk menjalankan file simulasi.ipynb sebagai batch job menggunakan Conda Environment User di Job Composer EWS:

1. Login ke EFIRO Web Service, buka menu Jobs > Job Composer, lalu buat ruang job dengan memilih menu New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" jika berjalan di compute node CPU atau "Conda Environment User (GPU)" jika memerlukan akselerasi GPU di compute node GPU, kemudian isi Job Name dan klik [Create New Job].
3. Siapkan file komputasi melalui menu [Edit Files] dengan batas upload 100MB per file; jika melebihi batas tersebut, gunakan Aplikasi SFTP yang diarahkan ke direktori Job Composer.
4. Karena batch job hanya menjalankan file Python (.py), Anda harus mengonversi file Notebook terlebih dahulu dengan membuka [Open Terminal], lalu jalankan perintah konversi berikut:
   $ jupytext --to py simulasi.ipynb
   Perintah ini akan menghasilkan file Python bernama simulasi.py.
5. Lakukan pengecekan kelengkapan environment dengan membuka [Open Terminal] kembali, lalu aktifkan modul pyload user yang akan digunakan menggunakan perintah:
   $ pyl load [nama-modul]
6. Jalankan pengecekan paket pada file Python yang akan dijalankan dengan perintah:
   $ pycheck simulasi.py
7. Perhatikan output hasil pengecekan: jika muncul "READY TO GO!" berarti file siap dijalankan, sedangkan jika muncul "MISSING PACKAGE!", instal package yang kurang sesuai panduan output (contoh: $ pip install --user pyusb).
8. Isi kolom submit script untuk menulis alur jalannya job dengan klik [Open Editor], ikuti petunjuk NOTES di dalamnya, lalu klik [Save] untuk menyimpan script.
9. Jalankan job dengan menekan tombol [Submit].
10. Pantau status dan output job dengan melakukan refresh halaman Job Composer; status dapat berupa Not Submitted, Queue, Running, Completed, atau Failed.
11. Jika job dalam status Running, Anda dapat memantau output terminal di kolom Folder Contents yang akan menampilkan file result-jobID.txt dan error-log-status-jobID.txt.
12. Untuk mengunduh file output, gunakan opsi menu [Edit Files] atau arahkan Aplikasi SFTP ke direktori Job Composer.

[Q30/69] Apa bedanya Effective Core Hour dengan Actual Core Hour di dalam sistem ALELEON?
------------------------------------------------------------
Berikut adalah perbedaan antara Effective Core Hour dan Actual Core Hour berdasarkan sistem ALELEON:

1. **Effective Core Hour Remaining** adalah sisa kuota core hour efektif yang menjadi basis Slurm ALELEON untuk menjalankan job, di mana nilainya berubah tergantung status job yang sedang berjalan atau tidak.
2. **Actual Core Hour Usage** adalah penggunaan core hour sesungguhnya (realtime) yang terjadi saat job berjalan dan menjadi dasar perhitungan biaya penggunaan ALELEON.

Secara spesifik, ketika tidak ada job yang berjalan, nilai *Effective Core Hour Remaining* sama dengan sisa kuota *Actual Core Hour*. Namun, ketika ada job yang sedang berjalan atau tertahan dalam antrian, *Effective Core Hour Remaining* dihitung berdasarkan sisa kuota *Total Core Hour* (yang mencakup *Booked Core Hour*), sedangkan *Actual Core Hour Usage* hanya menghitung waktu yang benar-benar telah digunakan oleh job tersebut.

[Q33/69] Saya submit 3 batch job berturut-turut. Job 1 pakai 64 CPU. Job 2 pakai 32 CPU. Kenapa saat saya submit Job 3 yang butuh 64 CPU, statusnya malah PENDING dengan tulisan QOSMaxCpuPerUserLimit, padahal node epyc masih banyak yang kosong?
------------------------------------------------------------
Status job Anda yang tertunda dengan alasan `QOSMaxCpuPerUserLimit` terjadi karena total alokasi CPU dari Job 1 dan Job 2 telah melebihi batas limit komputasi (Fair Usage Limit/LFU) yang diperbolehkan untuk akun perseorangan, bukan karena kekosongan node fisik.

Berdasarkan dokumen referensi, berikut adalah penjelasan dan langkah-langkah solusinya:

1. **Penyebab Terjadinya Pending**
   Sistem Slurm menahan job (status PENDING) apabila alokasi komputasi melebihi LFU atau jika akumulasi semua job user yang berjalan (status RUNNING) sudah menghabiskan sisa alokasi sesuai LFU. Meskipun node "epyc" kosong secara fisik, sistem menolak alokasi baru karena batasan logika per-user telah terpenuhi.
   
   *   **Batas Maksimal Akun Perseorangan:** Sesuai tabel "Berlaku untuk Akumulasi Semua Job Komputasi", limit maksimal CPU untuk akun perseorangan adalah **128**.
   *   **Analisis Kasus Anda:** Jika Job 1 (64 CPU) dan Job 2 (32 CPU) sedang berjalan bersamaan, total alokasi aktif adalah 96 CPU. Jika Anda mencoba menambah Job 3 sebesar 64 CPU, total menjadi 160 CPU, yang jelas melebihi batas 128 CPU. Atau, jika Job 1 dan 2 sudah selesai tetapi Job 3 masih menunggu alokasi yang memenuhi syarat LFU, maka sistem akan menunggu hingga ada ruang sesuai limit tersebut.

2. **Langkah Cek Awal**
   Untuk memastikan apakah alokasi job Anda melebihi Fair Limit, jalankan perintah berikut di terminal:
   $ squeue -lu $USER
   
   Cari nomor ID job yang statusnya PENDING atau RUNNING, lalu bandingkan dengan limit yang tersedia. Anda juga dapat melihat detail limit dan penggunaan saat ini dengan menjalankan tool:
   $ sausage menu sfair
   
   Perhatikan kolom **Fair Limit** dan **Current Usage** pada output tersebut.

3. **Solusi yang Tersedia**
   Jika hasil cek menunjukkan bahwa kebutuhan CPU melebihi limit, Anda memiliki dua opsi solusi:
   
   *   **Opsi 1: Batalkan dan Revisi Job**
       Batalkan salah satu job yang sedang berjalan atau antri, revisi jumlah CPU pada submit script agar total tidak melebihi limit, lalu jalankan ulang.
       
       Cara membatalkan job melalui terminal:
       1. Jalankan perintah: `$ squeue -lu $USER`
       2. Cari no. ID job yang ingin dibatalkan.
       3. Jalankan perintah: `$ scancel [job-ID]`
          *(Contoh membatalkan job ID 231: `$ scancel 231`)*
       
       Setelah dibatalkan, ubah parameter `#SBATCH --ntasks` pada submit script Job 3 atau job lainnya agar total alokasi tidak melebihi 128 CPU, kemudian jalankan kembali dengan perintah `$ sbatch [nama-submit-script]`.
   
   *   **Opsi 2: Ajukan Pembukaan Limitasi Sementara**
       Jika perhitungan Anda valid namun tetap terkendala, Anda dapat mengajukan pembukaan limitasi sementara kepada tim support admin. Tim admin akan memutuskan permintaan Anda menyesuaikan ketersediaan sumber daya komputasi dan antrian job pengguna lain di ALELEON Supercomputer.

    • Kode Reason ALELEON Supercomputer  (https://wiki.efisonlt.com/wiki/Kode_Reason_ALELEON_Supercomputer)
      💡 Why: Sumber ini relevan karena menjelaskan cara membaca kode alasan (reason) pada status job pending, termasuk `QOSMaxCpuPerUserLimit`, yang menjadi kunci identifikasi masalah.
    • Troubleshooting dan Support → A1. Daftar Kode Reason ALELEON Supercomputer  (https://wiki.efisonlt.com/wiki/Troubleshooting_dan_Support)

[Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Untuk membuat Conda Environment di ALELEON, Anda perlu membuka terminal melalui menu **Apps > Aleleon Shell Access** atau menggunakan **Login SSH**, lalu aktifkan modul Anaconda yang sesuai dengan perintah `$ module load Anaconda3/[versi]` (contoh: `$ module load Anaconda3/2024.02-1`). Setelah itu, buat environment kosong dengan Python versi tertentu menggunakan perintah `$ conda create -n [nama-conda-env] python=[versi-python]`, misalnya `$ conda create -n tensor python=3.11`.

Berikut adalah langkah-langkah lengkapnya berdasarkan dokumen referensi:

1. Buka terminal ALELEON dengan opsi Login SSH atau klik menu Apps > Aleleon Shell Access pada EFIRO Web Service.
2. Aktifkan modul Anaconda 3 yang diinginkan, misalnya untuk versi terbaru gunakan `$ module load Anaconda3/2025.06-1` atau versi stabil lainnya seperti `$ module load Anaconda3/2024.02-1`.
3. Buat Conda Env kosong dengan format perintah `$ conda create -n [nama-env] python=[versi-python]`, contohnya `$ conda create -n skripsi python=3.12`.
4. Aktifkan Conda Env yang baru dibuat dengan perintah `$ source activate [nama-env]`.
5. Selanjutnya, buat modul pyload untuk aktivasi Conda Env berikutnya dengan menjalankan perintah `(env) $ pyl mod` dan ikuti alur script yang muncul.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Untuk menjalankan sesi Jupyter menggunakan Conda Environment sendiri di ALELEON, Anda perlu mengikuti langkah persiapan dan konfigurasi formulir berikut ini:

1.  **Persiapan Conda Environment**
    Pastikan Anda telah membuat modul `pyload` untuk conda environment yang Anda miliki. Aktifkan terminal melalui menu **Clusters > Aleleon Shell Access** atau via **Login SSH**, lalu muat modul Anaconda dan aktifkan environment Anda dengan perintah:
    $ module load Anaconda3/2024.02-1
    $ pyl load [nama-modul]
    
    Jika Anda belum memiliki nama modul, cek ketersediaan dengan perintah `$ pyl list` pada bagian USER-DEFINED atau lihat detail info modul dengan `$ pyl info [nama-modul]`.

2.  **Instalasi Package (Opsional tapi Disarankan)**
    Sebelum memulai sesi Jupyter, disarankan menginstal package yang diperlukan agar efisien. Jalankan perintah berikut di dalam environment yang sudah aktif:
    (env) $ pip install jupyter notebook
    
    Untuk kebutuhan GPU, pastikan Anda menggunakan package CUDA versi >=11 sesuai spesifikasi hardware RTX 3090 yang tersedia.

3.  **Mengisi Formulir Sesi Jupyter**
    Login ke web **EFIRO**, klik aplikasi **Jupyter** (atau menu **Apps > Jupyter**), lalu isi formulir dengan panduan berikut:
    *   **Choose Jupyter version**: Pilih preferensi Anda antara **Jupyter Lab** atau **Notebook**.
    *   **Partition**: Pilih **torti** untuk interactive node CPU atau **tilla** untuk interactive node GPU.
    *   **Number of CPU thread(s)**: Masukkan jumlah core CPU (range 2 - 16).
    *   **Amount of memory/RAM (GB)**: Masukkan alokasi RAM (range 1 - 55).
    *   **Number of GPUs**: Isi angka **1** jika memilih partisi tilla.
    *   **Number of hours**: Tentukan waktu maksimal sesi (range 1 - 24).
    *   **Choose how to launch Jupyter**: Pilih **Anaconda/Mamba**.
    *   **Choose Anaconda/Mamba version**: Pilih **Anaconda 3 2024.02-1**.
    *   **Conda/Mamba environment directory**: Isi dengan nama conda environment user Anda (misalnya **tensor**) atau nama modul pyload yang dibuat.
    *   **Software Modules**: Kosongkan kecuali Anda memuat modul tambahan seperti CUDA (catatan: conda env 'tensor' tidak perlu memuat modul CUDA).
    *   **Environment setup**: Kosongkan jika tidak ada variabel khusus.
    *   **Additional Jupyter arguments**: Kosongkan jika tidak ada argumen tambahan.
    *   **Email address**: Isi jika ingin menerima notifikasi status mulai dan selesai.

4.  **Memulai dan Mengakses Sesi**
    Klik tombol **Launch** untuk memulai sesi. Anda akan diarahkan ke halaman **My Interactive Sessions**; tunggu hingga status berubah menjadi **Running**. Setelah status Running, klik tombol **Connect to Jupyter** untuk mengakses sesi. Sistem akan mulai menghitung Core Hour saat koneksi dilakukan. Gunakan kernel **Python 3 (ipykernel)** untuk sesi Notebook.

Sesi Jupyter akan terus berjalan selama waktu sesi masih tersedia, bahkan jika koneksi internet terputus atau Anda keluar dari web EFIRO. Anda dapat kembali masuk ke sesi yang sedang berjalan melalui menu **My Interactive Sessions**.

[Q66/69] Apa perbedaan antara menjalankan batch job via Job Composer EWS dan via terminal Slurm?
------------------------------------------------------------
Berikut adalah perbedaan utama antara menjalankan batch job melalui **Job Composer EWS** dan **Terminal Slurm** berdasarkan dokumen referensi:

1.  **Antarmuka dan Metode Login**
    *   **Job Composer EWS**: User login ke **[ EWS ]** atau **[ EFIRO Web Service ]**, lalu membuka menu **Jobs > Job Composer**. Proses dilakukan secara grafis melalui browser tanpa perlu akses terminal Linux langsung untuk langkah awal.
    *   **Terminal Slurm**: User harus menggunakan terminal Linux dengan opsi **[ Login SSH ]** atau login ke **[ EFIRO Web Service ]** pada menu **Clusters > Aleleon Shell Access**.

2.  **Persiapan File Komputasi**
    *   **Job Composer EWS**: File disiapkan via menu **[Edit Files]** di dalam antarmuka web. Terdapat batasan upload **100MB per file**; jika lebih, gunakan **[ Aplikasi SFTP ]** yang diarahkan ke direktori Job Composer.
    *   **Terminal Slurm**: File disiapkan di direktori **HOME** user menggunakan **[ Aplikasi SFTP ]** (login via EWS menu **Files > Home Directory**) atau transfer manual via SSH, dengan batas upload **100MB per file** untuk Home Directory.

3.  **Pembuatan Script Submit**
    *   **Job Composer EWS**: User mengklik tombol **[Open Editor]** di dalam ruang job untuk mengisi script submit. User mengikuti petunjuk **NOTES** dan alur script yang ada di dalamnya, kemudian menyimpannya dengan klik **[Save]**.
    *   **Terminal Slurm**: User membuat file script sendiri dengan format **.sh** (contoh: `submit.sh`) menggunakan editor teks di terminal. Script harus dimulai dengan header **#!/bin/bash -l** dan berisi konfigurasi SBATCH serta perintah pemanggilan modul (misalnya `pyl load` atau `ml load`) secara manual sesuai template.

4.  **Eksekusi Job**
    *   **Job Composer EWS**: Job dijalankan dengan mengklik tombol **[Submit]** pada halaman Job Composer.
    *   **Terminal Slurm**: Job dijalankan dengan mengetik perintah **$ sbatch [nama-submit-script]** (contoh: `$ sbatch submit.sh`).

5.  **Pemantauan Status**
    *   **Job Composer EWS**: Status dipantau dengan me-**refresh** halaman Job Composer. Daftar status meliputi *Not Submitted*, *Queue*, *Running*, *Completed*, dan *Failed*. Output dilihat di kolom **Folder Contents** (file `result-jobID.txt` dan `error-jobID.txt`).
    *   **Terminal Slurm**: Status dipantau menggunakan perintah **$ squeue -ul $USER**. Status *PENDING* ditandai dengan kode **PD**, dan *RUN* ditandai dengan kode **R**. Output dapat dilihat menggunakan perintah seperti `ls`, `cat`, `more`, atau `tail`.

6.  **Penghentian Job**
    *   **Job Composer EWS**: Jika ingin menghentikan job saat berjalan, user cukup mengklik tombol **[Stop]** pada antarmuka web.
    *   **Terminal Slurm**: Untuk menghentikan job, user harus menjalankan perintah **$ scancel [job-ID]** di terminal.

7.  **Fitur Khusus Python/Notebook**

[Q67/69] Bagaimana langkah lengkap membuat conda env baru dan modul pyload dari awal?
------------------------------------------------------------
Berikut adalah langkah lengkap untuk membuat conda environment baru dan modul pyload di ALELEON berdasarkan dokumen referensi:

1.  Buka terminal ALELEON melalui opsi **Login SSH** atau masuk ke menu **Clusters > Aleleon Shell Access** pada layanan **EFIRO Web Service**.
2.  Aktifkan modul Lmod Conda yang diinginkan, misalnya dengan perintah `$ ml Anaconda3/2025.06-1` (disarankan versi terbaru) atau `$ ml Mamba/23.11.0-0`. Hindari menggunakan `Anaconda3/2024.06-1` karena adanya laporan bug.
3.  Buat conda environment baru dengan format perintah berikut sesuai pilihan package manager Anda:
    *   Untuk Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
    *   Untuk Mamba: `$ mamba create -n [nama-env] python=[versi-python]`
    *   Contoh perintah jika menggunakan Anaconda dengan nama "skripsi" dan Python 3.12: `$ conda create -n skripsi python=3.12`
4.  Aktifkan conda environment yang baru dibuat dengan perintah `$ source activate [nama-env]`.
5.  Setelah lingkungan aktif, buat modul pyload untuk aktivasi otomatis di masa depan dengan menjalankan perintah `(env) $ pyl mod`, lalu ikuti alur script yang muncul dengan memilih menu no.1.

[Q68/69] Apa saja status job di squeue dan artinya masing-masing?
------------------------------------------------------------
Berikut adalah daftar status job (ST / STATE) yang ditampilkan oleh perintah `squeue` beserta artinya:

1. PD (PENDING) - Job mengantri dan belum berjalan; jika tertahan dalam waktu lama, disarankan untuk melihat bagian Troubleshooting 1.
2. R (RUN) - Job sedang berjalan saat ini.
3. CG (COMPLETING) - Job telah selesai dan sedang dalam proses pembersihan (clean-up).
4. CA (CANCELED) - Job dibatalkan secara manual oleh user.
5. PR (PREEMPETED) - Job dibatalkan oleh tim admin, dengan notifikasi dan alasan dikirimkan melalui email support.
6. S (SUSPENDED) - Job ditahan sementara oleh tim admin, dengan notifikasi dan alasan dikirimkan melalui email support.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah panduan lengkap cara mengisi formulir Jupyter di EWS untuk Conda Environment User berdasarkan dokumen referensi:

1.  Login ke **EFIRO Web Service** dan buka menu **Interactive Apps > Jupyter**.
2.  Pada bagian **Number of CPU threads**, pilih jumlah core CPU sesuai kebutuhan.
3.  Pada bagian **Use GPU for the job**, pilih **no** (tanpa GPU) atau **yes** (1x RTX 3090) sesuai kebutuhan komputasi.
4.  Pada bagian **Number of hours**, isi waktu maksimal sesi Jupyter yang diinginkan.
5.  Pada bagian **Choose package to run**, pilih opsi **User provided**.
6.  Pada bagian **Choose how to launch Jupyter**, pilih **Anaconda / Mamba**.
7.  Pada bagian **Choose Anaconda/Mamba version & User provided Conda/Mamba environment**, masukkan informasi modul Lmod conda dan nama conda env user.
    *   Untuk membantu pengisian, Anda dapat membuka menu **Clusters > Aleleon Shell Access**, lalu jalankan perintah `$ pyl info [nama-modul]` untuk melihat info modul pyload user.
    *   Gunakan nilai **Conda Lmod module** sebagai pilihan versi Anaconda/Mamba.
    *   Gunakan nilai **Env name** sebagai nama Conda/Mamba environment yang disediakan.
