[Q7/69] Saya mau cek sisa kuota core hour saya. Perintah sausage apa yang harus diketik?

1. Buka shell atau terminal ALELEON.
2. Bagi pengguna yang login melalui EWS, buka menu Clusters > Aleleon Shell Access.
3. Bagi pengguna di EWS - app Job Composer, klik menu yang tersedia untuk akses shell.
4. Jalankan perintah `$ sausage` untuk membaca penggunaan core hour user.
5. Pilih menu `sfair` untuk membaca info Limit Fair Usage (kuota alokasi komputasi user) dan penggunaannya.

[Q12/69] Bagaimana cara membatalkan/menghentikan job yang berstatus PENDING di terminal?
------------------------------------------------------------
Untuk menghentikan atau membatalkan job yang sedang berjalan (termasuk status PENDING), Anda dapat menjalankan perintah `scancel` diikuti dengan ID job tersebut di terminal.

Berikut adalah langkah-langkahnya:
1. Lihat daftar status job dan ambil Job ID-nya menggunakan perintah `$ squeue -ul $USER`.
2. Jalankan perintah pembatalan dengan format `$ scancel [job-ID]`, misalnya untuk membatalkan job ID 231 gunakan perintah `$ scancel 231`.

Perintah ini akan mengubah status job menjadi CANCELED (dibatalkan oleh user).

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Untuk menjalankan file simulasi.ipynb Anda sebagai batch job di Job Composer EWS menggunakan Conda Environment User, ikuti langkah-langkah berikut secara berurutan:

1. Login ke EFIRO Web Service, buka menu Jobs > Job Composer, lalu buat ruang job baru dengan memilih menu New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" untuk berjalan di compute node CPU atau "Conda Environment User (GPU)" jika membutuhkan akselerasi GPU, kemudian isi kolom Job Name dan klik [Create New Job].
3. Siapkan file komputasi melalui menu [Edit Files] dengan upload file simulasi.ipynb (perhatikan limit upload 100MB per file).
4. Karena batch job hanya menjalankan file Python (.py), buka [Open Terminal] dan lakukan konversi file Notebook menjadi file Python dengan perintah: $ jupytext --to py simulasi.ipynb.
5. Buka kembali [Open Terminal], aktifkan modul pyload user yang akan digunakan dengan perintah $ pyl load [nama-modul], lalu lakukan pengecekan kelengkapan package dengan perintah $ pycheck simulasi.py.
6. Jika output menunjukkan MISSING PACKAGE, instal package yang kurang sesuai panduan output (contoh: $ pip install --user pyusb) dengan ketentuan user hanya dapat menginstal package yang belum ada dan tidak dapat update package bawaan.
7. Isi kolom submit script dengan klik [Open Editor], ikuti petunjuk NOTES dan alur script di dalamnya, lalu klik [Save] untuk menyimpan script.
8. Jalankan job dengan mengklik tombol [Submit].
9. Pantau status dan output job dengan refresh halaman Job Composer; status Running memungkinkan pemantauan output di kolom Folder Contents (result-jobID.txt dan error-jobID.txt) serta utilisasi CPU dan memori melalui menu Monitoring > Grafana pada Jobs > Active Jobs.

[Q32/69] (Troubleshooting) Saya submit job GROMACS tapi selalu gagal dengan pesan error Invalid syntax. Di script saya menulis #SBATCH --mem= 64 GB. Apa yang salah dari tulisan saya?
------------------------------------------------------------
Masalah pada penulisan perintah `#SBATCH --mem= 64 GB` Anda terletak pada adanya spasi yang tidak seharusnya ada di sekitar tanda sama dengan dan di antara nilai serta satuan.

Berdasarkan dokumen referensi, format penulisan SBATCH yang benar **tidak boleh** memiliki spasi di parameter, tanda sama dengan (`=`), maupun nilai. Berikut adalah perbandingan kesalahan dan kebenaran sesuai panduan:

1.  **Penulisan Salah (Contoh dari dokumen):**
    *   `#SBATCH --mem= 64 GB` (terdapat spasi setelah `=`)
    *   `#SBATCH --mem= 64GB` (terdapat spasi sebelum `GB`)
    *   `#SBATCH --time =01:00:00` (terdapat spasi sebelum `=`)

2.  **Penulisan Benar:**
    *   Format harus digabung tanpa spasi: `#SBATCH --mem=64GB`
    *   Contoh penulisan waktu yang benar juga tanpa spasi: `#SBATCH --time=01:00:00`

Anda perlu mengubah baris tersebut menjadi `#SBATCH --mem=64GB` agar sintaksnya valid dan job dapat berjalan.

[Q33/69] Saya submit 3 batch job berturut-turut. Job 1 pakai 64 CPU. Job 2 pakai 32 CPU. Kenapa saat saya submit Job 3 yang butuh 64 CPU, statusnya malah PENDING dengan tulisan QOSMaxCpuPerUserLimit, padahal node epyc masih banyak yang kosong?
------------------------------------------------------------
Halo, masalah status **PENDING** dengan kode alasan **QOSMaxCpuPerUserLimit** pada Job 3 Anda terjadi karena total alokasi CPU yang Anda ajukan telah melebihi batas maksimal (limit) yang ditetapkan untuk pengguna perorangan atau grup akun institusi tersebut.

Berdasarkan dokumen referensi, berikut adalah penjelasan dan langkah-langkah penanganan yang tersedia:

1.  **Analisis Penyebab Limit**
    Kode **QOSMaxCpuPerUserLimit** mengindikasikan bahwa permintaan komputasi Anda (Job 3 yang membutuhkan 64 CPU) ditambah dengan sisa alokasi dari job lain yang sedang berjalan atau antri telah melampaui **Fair Usage Limit (LFU)** user Anda.
    *   Jika Job 1 (64 CPU) dan Job 2 (32 CPU) masih berjalan atau antri, total penggunaan CPU Anda mungkin sudah mencapai batas maksimal yang diizinkan sistem untuk satu sesi atau periode tertentu.
    *   Dokumen menyebutkan bahwa pada user perseorangan atau grup, jika ada job lain milik user atau sesama grup yang berjalan dan menyisakan alokasi komputasi yang tidak mencukupi untuk job antri baru, maka job tersebut akan tertahan.

2.  **Langkah Cek Awal Limit Komputasi**
    Untuk memverifikasi apakah limit Anda telah terlampaui, jalankan perintah berikut melalui terminal untuk melihat besar limit komputasi (LFU) di kolom **Fair Limit**:
    $ sausage sfair

    Pada output tersebut, perhatikan kolom **ntasks** (yang padanannya adalah `ntasks * cpus-per-task` untuk CPU). Bandingkan nilai ini dengan jumlah CPU yang Anda butuhkan untuk Job 3.

3.  **Opsi Penyelesaian**
    Jika hasil cek menunjukkan bahwa Anda memiliki alokasi yang berlebih atau melebihi limit, terdapat dua opsi penyelesaian sesuai prosedur:

    **Opsi 1: Batalkan job bersangkutan, revisi jumlah SBATCH, dan jalankan ulang**
    Anda dapat membatalkan salah satu job yang sedang berjalan atau mengantri (misalnya Job 1 atau Job 2) untuk membebaskan slot CPU agar Job 3 bisa berjalan.
    *   **Melalui EFIRO Web Service:**
        1. Buka menu **Apps > Active Jobs**.
        2. Ubah opsi **All Jobs** ke **Your Jobs**.
        3. Cari job yang akan dibatalkan.
        4. Pada kolom Action, klik icon **(Delete Job)**.
    *   **Melalui Terminal:**
        1. Jalankan perintah untuk mencari ID job PENDING atau RUNNING:
           $ squeue -lu $USER
        2. Cari nomor ID job yang ingin dibatalkan.
        3. Jalankan perintah pembatalan (contoh membatalkan job ID 231):
           $ scancel [job-ID]

    Setelah membatalkan job yang memakan sumber daya, Anda perlu merevisi script batch (SBATCH) untuk Job 3 atau menunggu hingga job lain selesai, lalu jalankan kembali Job 3.

    **Opsi 2: Mengajukan pembukaan limitasi sementara**
    Jika Anda yakin perhitungan kebutuhan komputasi Anda valid dan mendesak, Anda dapat mengajukan permohonan pembukaan limitasi sementara kepada tim admin support. Tim admin akan memutuskan permintaan Anda menyesuaikan ketersediaan sumber daya komputasi dan antrian job pengguna lain di ALELEON Supercomputer.

4.  **Catatan Penting Mengenai Ketersediaan Node**
    Meskipun node epyc terlihat kosong, sistem Slurm tidak hanya melihat ketersediaan fisik node, tetapi juga mematuhi kebijakan **QoS (Quality of Service)** seperti **QOSMaxCpuPerUserLimit**. Artinya, meskipun hardware tersedia, sistem menolak penjadwalan job baru jika pengguna tersebut telah melanggar batas alokasi CPU per pengguna yang telah dikonfigurasi oleh administrator.

[Q34/69] (Troubleshooting) Job saya berstatus PD dengan alasan AssocMaxWallDurationPerJobLimit. Di script saya menulis #SBATCH --time=4-00:00:00. Akun saya adalah akun perseorangan biasa. Mengapa tertahan?
------------------------------------------------------------
Status job Anda tertahan (PENDING) dengan alasan `AssocMaxWallDurationPerJobLimit` karena durasi waktu yang Anda ajukan melebihi batas maksimal yang diperbolehkan untuk akun perseorangan biasa.

Berdasarkan dokumen referensi, berikut adalah rincian batasan dan langkah yang harus Anda lakukan:

1.  **Batas Waktu Akun:** Untuk akun uji coba atau akun standar yang disebutkan dalam template submit script Conda Environment versi CPU dan GPU, batas maksimal walltime (waktu limit komputasi) adalah **24 jam**.
    *   Format penulisan maksimalnya adalah `24:00:00`.
    *   Penulisan `4-00:00:00` (yang setara dengan 96 jam) melanggar aturan ini sehingga SLURM menahan job tersebut.

2.  **Cara Menentukan Batas Alokasi:** Dokumen menyarankan agar user tidak mengarang angka sendiri, melainkan menentukan rentang nilai alokasi komputasi berdasarkan ketentuan metrik (SLA) ALELEON dengan cara menjalankan perintah berikut:
    $ sausage
    Pilih menu slimit -> job type: Batch job

3.  **Solusi Perbaikan Script:** Anda perlu mengubah baris `#SBATCH --time` di dalam script submit Anda menjadi durasi yang sesuai dengan batas akun Anda (maksimal 24 jam).
    *   Contoh penulisan yang benar untuk batas maksimal 24 jam:
        #SBATCH --time=24:00:00
    *   Atau format alternatif:
        #SBATCH --time=1-00:00:00

Setelah mengubah nilai waktu di script submit menjadi tidak lebih dari 24 jam, silakan jalankan kembali perintah `sbatch [nama-submit-script]` untuk mengirim job.

[Q35/69] Saya mau menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array. Setiap simulasi butuh 4 core CPU dan 8GB RAM. Di script, apakah saya harus menulis --cpus-per-task=40 atau --cpus-per-task=4?
------------------------------------------------------------
Untuk menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan Slurm Array dengan setiap simulasi membutuhkan 4 core CPU, Anda harus menulis `#SBATCH --cpus-per-task=4` pada script submit.

Berdasarkan dokumen referensi, parameter `--cpus-per-task` mendefinisikan jumlah core thread CPU yang digunakan oleh **setiap** array job (satu simulasi), bukan total keseluruhan. Total alokasi CPU dihitung secara otomatis oleh sistem sebagai hasil perkalian antara jumlah array dan nilai `cpus-per-task`.

Sebagai contoh sesuai dokumentasi:
1. Jika Anda menetapkan `#SBATCH --array=1-10` untuk 10 simulasi.
2. Dan menetapkan `#SBATCH --cpus-per-task=4` untuk kebutuhan per simulasi.
3. Maka total alokasi CPU akan menjadi 10 * 4 = 40 core, dan total memori RAM akan menjadi 10 * 8GB = 80GB.

Menulis `--cpus-per-task=40` akan salah karena itu berarti setiap dari 10 simulasi tersebut akan mencoba mengambil 40 core, sehingga total yang dibutuhkan menjadi 400 core, yang tidak sesuai dengan kebutuhan Anda.

[Q37/69] Saya butuh komputasi memori raksasa sebesar 350 GB untuk satu aplikasi yang non-MPI (tidak bisa dibagi ke banyak node). Partisi apa yang WAJIB saya gunakan agar tidak error kehabisan memori?

4. Jika Anda tetap ingin mencoba menggunakan partisi epyc-jumbo, perhatikan bahwa parameter `--mem` dalam SBATCH berlaku per node, bukan total keseluruhan, sesuai instruksi: `#SBATCH --mem=[alokasi memori per node]`. [Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Berikut adalah langkah-langkah resmi untuk membuat Conda Environment di ALELEON berdasarkan dokumentasi:

1. Buka terminal ALELEON melalui opsi Login SSH atau menu EFIRO Web Service pada Clusters > Aleleon Shell Access.
2. Aktifkan modul Lmod conda yang sesuai dengan kebutuhan, misalnya versi terbaru Anaconda 3 dengan perintah `$ ml Anaconda3/2025.06-1` (atau gunakan `$ ml Anaconda3/2024.02-1` untuk Python 3.11).
3. Buat environment kosong menggunakan format perintah `$ conda create -n [nama-env] python=[versi-python]`, contohnya `$ conda create -n tensor python=3.11`.
4. Aktifkan environment yang baru dibuat dengan perintah `$ source activate [nama-env]` dan selanjutnya buat modul pyload untuk manajemen komputasi dengan menjalankan `pyl mod`.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Untuk menjalankan sesi Jupyter menggunakan Conda Environment buatan Anda sendiri di ALELEON, ikuti langkah-langkah berikut berdasarkan panduan sistem:

[Q68/69] Apa saja status job di squeue dan artinya masing-masing?

1. R (RUN): Job sedang berjalan.
2. PD (PENDING): Job tertahan, pengguna perlu melihat NODELIST(REASON) untuk alasan penundaan.
3. CG (COMPLETING): Job telah selesai dan sedang dalam proses clean-up.
4. CA (CANCELED): Job dibatalkan oleh user.
5. PR (PREEMPETED): Job dibatalkan oleh admin, dengan alasan yang dikabarkan melalui email.
6. S (SUSPENDED): Job ditahan oleh admin, dengan alasan yang dikabarkan melalui email.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah langkah-langkah lengkap untuk mengisi formulir memulai sesi Jupyter di EWS sesuai dengan panduan sistem:

1.  Login ke web EFIRO ALELEON Supercomputer melalui menu **Apps > Jupyter** atau klik aplikasi **Jupyter** di homepage.
2.  Pilih versi Jupyter yang diinginkan, yaitu **Jupyter Lab** atau **Notebook**.
3.  Pada bagian **Partition**, pilih **torti** untuk interactive node CPU atau **tilla** untuk interactive node GPU.
4.  Isi jumlah **Number of CPU thread(s)** dalam rentang **2 - 16**.
5.  Isi **Amount of memory/RAM (GB)** dalam satuan GB dengan rentang **1 - 55**.
6.  Jika memilih partition **tilla**, isi **Number of GPUs** sebanyak **1**.
7.  Tentukan **Number of hours** untuk waktu maksimal sesi Jupyter dalam rentang **1 - 24**.
8.  Pada bagian **Choose how to launch Jupyter**, pilih opsi **Anaconda/Mamba**.
9.  Pada **Choose Anaconda/Mamba version**, pilih **Anaconda 3 2024.02-1**.
10. Di kolom **Conda/Mamba environment directory**, isi nama conda env yang dibuat user (contoh: **tensor**), kecuali jika ingin menggunakan lingkungan lain.
11. Pada bagian **Software Modules**, isi nama modul CUDA hanya jika diperlukan dan tidak menggunakan conda env 'tensor' (conven 'tensor' tidak perlu memuat modul CUDA).
12. Isi kolom **Environment setup** dan **Additional Jupyter arguments** hanya jika ada variabel atau argumen tambahan yang dibutuhkan.
13. Isi **Email address** jika ingin menerima notifikasi status mulai dan selesainya sesi Jupyter.
14. Klik tombol **Launch** untuk memulai sesi, lalu tunggu hingga status berubah menjadi **Running** sebelum mengklik **Connect to Jupyter**.
