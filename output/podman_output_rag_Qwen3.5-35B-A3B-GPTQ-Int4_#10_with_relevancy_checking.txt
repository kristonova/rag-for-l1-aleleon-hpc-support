1.  **Definisi dan Fungsi Utama**
    *   **Actual Core Hour Usage** adalah penggunaan core hour sesungguhnya (realtime) yang menjadi basis biaya penggunaan ALELEON.

    *   Ketika **tidak ada job berjalan**, Effective Core Hour Remaining sama dengan sisa kuota Actual Core Hour.
    *   Ketika **ada job berjalan**, Effective Core Hour Remaining dihitung berdasarkan sisa kuota Total Core Hour (yang mencakup Booked Core Hour), bukan hanya Actual Core Hour.

3.  **Contoh Perhitungan**
    *   Jika user memiliki kuota 500 CCH dan menjalankan job dengan alokasi 32 core CPU selama 10 jam:
        *   Manajemen Slurm mengalokasikan "Booked Core Hour" sebesar 320 CCH (32 core x 10 jam).
        *   Saat ini, **Total Core Hour Usage** tercatat sebesar 320 CCH.
        *   **Effective Core Hour Remaining** menjadi 180 CCH (500 CCH - 320 CCH).
        *   Namun, **Actual Core Hour Usage** pada saat itu mungkin masih 0 atau baru bertambah sesuai durasi waktu nyata yang sudah berlalu (misalnya setelah 1 jam berjalan, Actual menjadi 32 CCH), namun sisa Effective tetap dihitung dari Total yang terbooked.

4.  **Kondisi Penahanan Job**
    *   Sistem akan menahan job jika kebutuhan core hour job lebih besar dari Effective Core Hour Remaining.
    *   Jika tidak ada job jalan, pengecekan dilakukan terhadap sisa kuota **Actual Core Hour**.

Berikut adalah langkah-langkah lengkap untuk membuat Conda Environment di ALELEON berdasarkan panduan resmi:

1. Buka terminal ALELEON melalui opsi Login SSH atau masuk ke menu Clusters > Aleleon Shell Access pada EFIRO Web Service.
2. Aktifkan modul Lmod conda yang sesuai dengan kebutuhan Anda menggunakan perintah berikut (pilih salah satu versi):

   *Catatan: Hindari penggunaan Anaconda3 2024.06-1 karena adanya laporan bug.*
3. Buat Conda Environment kosong dengan format perintah berikut (sesuaikan nama env dan versi Python):
   - Menggunakan Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
     Contoh: `$ conda create -n skripsi python=3.12`
   - Menggunakan Mamba: `$ mamba create -n [nama-env] python=[versi-python]`
   - Jika memiliki file YAML resep instalasi package:
     - Anaconda: `$ conda create -f [nama-file-yml].yml`
     - Mamba: `$ mamba create -f [nama-file-yml].yml`
4. Aktifkan Conda Environment yang baru dibuat dengan perintah:
   `$ source activate [nama-env]`
5. Setelah aktif, buat modul pyload untuk aktivasi environment di sesi berikutnya dengan menjalankan:
   `(env) $ pyl mod`
   Kemudian ikuti alur script yang muncul dengan memilih menu no.1.
