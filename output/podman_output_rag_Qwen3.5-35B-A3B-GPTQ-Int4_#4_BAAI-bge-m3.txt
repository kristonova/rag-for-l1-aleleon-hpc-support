[Q1/69] Berapa kapasitas RAM efektif per node di partisi epyc-jumbo?

1. Partisi epyc-jumbo memiliki 1 node.
2. Setiap node dilengkapi dengan 1x AMD EPYC 7702P (64 core / 128 thread, 2.0Ghz).

4. Interkoneksi yang digunakan adalah Mellanox RoCE 100Gbps.

[Q2/69] Berapa batas maksimal walltime (waktu komputasi) per job untuk golongan akun perseorangan?
------------------------------------------------------------
Berdasarkan dokumen referensi, batas maksimal walltime per job untuk golongan akun perseorangan adalah **3 hari (72 jam)**.

Jika Anda mencoba menjalankan job dengan alokasi waktu melebihi batas ini, manajemen Slurm akan menahan job tersebut dalam status PENDING. Berikut adalah rincian lengkapnya:

1. Limit maksimal walltime untuk Akun Perseorangan ditetapkan sebesar 3 hari atau setara dengan 72 jam.
2. Keterangan di dokumen menyatakan bahwa walltime adalah waktu maksimal menjalankan job dan manajemen Slurm akan menahan job jika menggunakan alokasi komputasi melebihi LFU ini.
3. Jika job Anda tertahan karena alasan `AssocMaxWallDurationPerJobLimit`, solusi yang disarankan adalah membatalkan job tersebut dan mengubah waktu SBATCH di bawah limit ALELEON.

[Q12/69] Bagaimana cara membatalkan/menghentikan job yang berstatus PENDING di terminal?
------------------------------------------------------------
Untuk membatalkan atau menghentikan job yang berstatus PENDING di terminal, Anda dapat menggunakan perintah `scancel` diikuti dengan ID job tersebut. Berikut adalah langkah-langkahnya:

1. Lihat status dan ID job Anda dengan menjalankan perintah `$ squeue -ul $USER`.
2. Identifikasi Job ID dari kolom yang ditampilkan pada hasil perintah `squeue`.
3. Jalankan perintah pembatalan dengan format `$ scancel [job-ID]`, contoh jika Job ID adalah 231 maka ketik `$ scancel 231`.

Perintah ini akan mengubah status job menjadi CA (CANCELED) yang berarti job dibatalkan oleh user.

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Untuk menjalankan file simulasi.ipynb sebagai batch job menggunakan Conda Environment User di Job Composer EWS, ikuti langkah-langkah berikut:

1. Login ke EFIRO Web Service, buka menu Jobs > Job Composer, lalu buat ruang job dengan memilih New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" untuk berjalan di compute node CPU atau "Conda Environment User (GPU)" jika memerlukan akselerasi GPU, kemudian isi Job Name dan klik [Create New Job].
3. Siapkan file komputasi via menu [Edit Files] dengan upload file simulasi.ipynb (limit upload 100MB per file; bila lebih gunakan Aplikasi SFTP).
4. Lakukan konversi file Notebook ke Python karena batch job hanya menjalankan file .py dengan membuka [Open Terminal] dan jalankan perintah:
   $ jupytext --to py simulasi.ipynb
5. Cek kelengkapan package conda environment dengan membuka [Open Terminal], aktifkan modul pyload user yang akan digunakan menggunakan perintah:
   $ pyl load [nama-modul]
6. Lakukan pengecekan pada file Python hasil konversi dengan perintah:
   $ pycheck simulasi.py
7. Perhatikan output dari pycheck: jika muncul "READY TO GO!" berarti file siap dijalankan, sedangkan jika muncul "MISSING PACKAGE!", instal package yang kurang sesuai panduan output (contoh: $ pip install --user pyusb).
8. Isi kolom submit script untuk menulis alur jalannya job dengan klik [Open Editor], ikuti petunjuk NOTES di dalamnya, lalu klik [Save].
9. Jalankan job dengan mengklik tombol [Submit].
10. Pantau status dan output job dengan refresh halaman Job Composer; status dapat berupa Not Submitted, Queue, Running, Completed, atau Failed.
11. Untuk melihat output terminal dan log status, buka kolom Folder Contents yang berisi file result-jobID.txt dan error-jobID.txt.
12. Jika ingin menghentikan job saat sedang berjalan, klik tombol [Stop]; jika ingin menjalankan ulang setelah selesai, klik kembali [Submit].

[Q30/69] Apa bedanya Effective Core Hour dengan Actual Core Hour di dalam sistem ALELEON?
------------------------------------------------------------
Effective Core Hour Remaining adalah sisa kuota yang menjadi basis Slurm ALELEON untuk menjalankan job, di mana nilainya berubah tergantung apakah ada job yang sedang berjalan atau tidak. Jika tidak ada job jalan, Effective Core Hour Remaining sama dengan sisa kuota Actual Core Hour, namun jika ada job berjalan, Effective Core Hour Remaining dihitung dari sisa kuota Total Core Hour (yang mencakup Booked Core Hour).

Sementara itu, Actual Core Hour Usage adalah penggunaan core hour sesungguhnya secara realtime yang menjadi dasar perhitungan biaya penggunaan ALELEON.

[Q33/69] Saya submit 3 batch job berturut-turut. Job 1 pakai 64 CPU. Job 2 pakai 32 CPU. Kenapa saat saya submit Job 3 yang butuh 64 CPU, statusnya malah PENDING dengan tulisan QOSMaxCpuPerUserLimit, padahal node epyc masih banyak yang kosong?
------------------------------------------------------------
Status PENDING dengan alasan `QOSMaxCpuPerUserLimit` pada Job 3 Anda terjadi karena total alokasi CPU yang diminta melebihi batas limit komputasi (Fair Usage Limit/LFU) per pengguna atau sisa alokasi LFU yang tersedia di sistem.

Berdasarkan dokumen referensi, berikut adalah penjelasan dan langkah-langkah yang harus Anda lakukan:

1.  **Penyebab Masalah**
    Sistem Slurm menahan job (status PENDING) apabila penggunaan alokasi komputasi melebihi LFU atau jika akumulasi semua job yang berjalan (termasuk Job 1 dan Job 2 yang sedang aktif) telah menghabiskan sisa alokasi sesuai LFU user tersebut. Meskipun node fisik kosong, sistem tidak mengizinkan eksekusi karena batasan kuota per pengguna telah tercapai.

2.  **Langkah Cek Awal**
    Jalankan tool `sausage` untuk mengetahui besar limit komputasi (LFU) dan sisa alokasi Anda saat ini:
    $ sausage
    Pilih menu `sfair` untuk melihat kolom "Fair Limit" dan "Current Usage".

3.  **Analisis Padanan SBATCH vs Fair Limit**
    Bandingkan alokasi CPU yang Anda minta di script submit dengan limit LFU Anda menggunakan padanan berikut:
    *   Jika di SBATCH Anda menggunakan `ntasks`, maka hitungan CPU adalah `ntasks * cpus-per-task`.
    *   Jika total alokasi job Anda (Job 1 + Job 2 + Job 3) melebihi nilai di kolom "Fair Limit", maka job baru akan tertahan.

4.  **Solusi yang Tersedia**
    Jika hasil cek menunjukkan ada alokasi yang melebihi limit, Anda memiliki dua opsi:
    
    *   **Opsi 1: Batalkan dan Revisi Job**
        Batalkan job yang berlebih (misalnya Job 3), revisi jumlah alokasi CPU di script SBATCH agar sesuai dengan sisa limit LFU, lalu jalankan ulang.
        Untuk membatalkan job via terminal, gunakan perintah:
        $ squeue -lu $USER
        Cari ID job yang PENDING, lalu jalankan:
        $ scancel [job-ID]
        
    *   **Opsi 2: Ajukan Pembukaan Limitasi Sementara**
        Ajukan pembukaan limitasi sementara ke support tim admin. Tim admin akan memutuskan permintaan Anda menyesuaikan ketersediaan sumber daya komputasi dan antrian job pengguna lain di ALELEON Supercomputer.

5.  **Informasi Batas Maksimal**
    Dokumen menyebutkan bahwa Akun Perseorangan memiliki batas maksimal CPU sebesar 128 core thread. Namun, batas spesifik untuk akun institusi atau uji coba berbeda (sesuai PKSPIAS atau 16 core). Pastikan total penggunaan Anda tidak melebihi batas maksimal akun Anda masing-masing.

[Q35/69] Saya mau menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array. Setiap simulasi butuh 4 core CPU dan 8GB RAM. Di script, apakah saya harus menulis --cpus-per-task=40 atau --cpus-per-task=4?
------------------------------------------------------------
Untuk menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan Slurm Array dengan setiap simulasi membutuhkan 4 core CPU, Anda harus menulis `#SBATCH --cpus-per-task=4` pada script submit.

Berdasarkan dokumen referensi, alokasi `--cpus-per-task` digunakan oleh **setiap** array job secara individual, bukan untuk total keseluruhan. Berikut adalah penjelasan rincinya:

1.  Dalam contoh penggunaan Slurm array pada dokumen, jika user hendak menjalankan 8 job array dengan masing-masing 2 core CPU, perintah yang digunakan adalah `#SBATCH --cpus-per-task=2`.
2.  Total alokasi CPU dihitung dengan rumus: `Total CPU = cpus-per-task * jumlah_array`. Jika Anda menulis `--cpus-per-task=4` dan `--array=1-10`, maka total alokasi menjadi 40 core (4 * 10), yang sesuai dengan kebutuhan Anda.
3.  Jika Anda menulis `--cpus-per-task=40`, maka setiap satu dari 10 simulasi tersebut akan mencoba mengambil 40 core, sehingga total alokasi menjadi 400 core, yang jauh melebihi kebutuhan per simulasi.

Jadi, konfigurasi yang benar dalam script adalah:
```bash
#SBATCH --array=1-10
#SBATCH --cpus-per-task=4
#SBATCH --mem=8GB
```

[Q39/69] Kenapa saat saya meminta alokasi #SBATCH --ntasks=7, sistem Slurm ALELEON akan membulatkannya menjadi 8 dan saya ditagih biaya untuk 8 core?
------------------------------------------------------------
Sistem Slurm ALELEON membulatkan alokasi core CPU ganjil menjadi genap ke atas untuk mengoptimalkan performa karena SMT (Simultaneous Multi-Threading) diaktifkan pada semua node, sehingga 2 thread dalam 1 core tidak boleh dipakai oleh 2 job berbeda.

Berdasarkan dokumen referensi, berikut adalah alasan dan mekanisme pembulatan tersebut:
1. ALELEON mengaktifkan SMT pada semua CPU compute node yang berjalan 2 thread per core.
2. Slurm dikonfigurasi agar 2 thread dalam 1 core CPU tidak dipakai oleh 2 job berbeda demi optimasi performa.
3. Oleh karena itu, jumlah alokasi CPU ganjil akan dibulatkan genap ke atas oleh Slurm, contohnya jumlah 7 akan menjadi 8.
4. Karena sistem mengalokasikan 8 core untuk menjalankan job Anda, maka biaya Core Hour (CCH) dihitung berdasarkan alokasi yang diberikan sistem, yaitu 8 core.

[Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Berikut adalah langkah-langkah resmi untuk membuat Conda Environment di ALELEON berdasarkan dokumentasi sistem:

1. Buka terminal ALELEON melalui opsi Login SSH atau menu Apps > Aleleon Shell Access pada EFIRO Web Service.
2. Aktifkan modul Anaconda yang sesuai dengan kebutuhan Anda, misalnya versi terbaru 2025.06-1 atau versi stabil lainnya seperti 2024.02-1 menggunakan perintah `module load` (contoh: `$ ml Anaconda3/2025.06-1`).
3. Buat environment kosong dengan menentukan nama dan versi Python yang diinginkan menggunakan perintah `$ conda create -n [nama-env] python=[versi-python]`.
4. Setelah environment dibuat, aktifkan lingkungan tersebut dengan perintah `$ source activate [nama-env]`.
5. Untuk memudahkan penggunaan di sesi komputasi selanjutnya, buat modul pyload khusus untuk environment tersebut dengan menjalankan perintah `(env) $ pyl mod` dan ikuti alur script yang muncul.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Berikut adalah langkah-langkah untuk menjalankan sesi Jupyter menggunakan Conda Environment yang telah Anda buat sendiri di ALELEON:

1.  **Persiapan Awal:** Pastikan Anda telah melengkapi instalasi package pada Conda Environment (conda env) yang akan digunakan sebelum memulai sesi, karena hal ini penting agar script berjalan tanpa error dan efisien dalam penggunaan Core Hour.
2.  **Login ke Web EFIRO:** Masuk ke web EFIRO ALELEON Supercomputer melalui menu Login.
3.  **Akses Aplikasi Jupyter:** Klik aplikasi "Jupyter" di homepage atau buka menu "Apps > Jupyter".
4.  **Isi Formulir Sesi Jupyter:** Lakukan pengisian formulir dengan ketentuan berikut sesuai dokumen referensi:
    *   **Choose Jupyter version:** Pilih preferensi Anda antara "Jupyter Lab" atau "Notebook".
    *   **Partition:** Pilih partisi sesuai kebutuhan komputasi interaktif, yaitu "torti" untuk CPU atau "tilla" untuk GPU.
    *   **Number of CPU thread(s):** Masukkan jumlah alokasi core CPU dengan rentang 2 - 16 (atau 1-32 untuk akun perseorangan).
    *   **Amount of memory/RAM (GB):** Masukkan jumlah RAM dalam satuan GB dengan rentang 1 - 55 (atau 1-64 untuk akun perseorangan).
    *   **Number of GPUs:** Jika memilih partisi "tilla", isi angka 1.
    *   **Number of hours:** Tentukan waktu maksimal sesi dengan rentang 1 - 24 jam (atau 1-72 jam untuk akun perseorangan).
    *   **Choose how to launch Jupyter:** Pilih opsi "Anaconda/Mamba".
    *   **Choose Anaconda/Mamba version:** Pilih versi "Anaconda 3 2024.02-1" (atau versi terbaru yang tersedia).
    *   **Conda/Mamba environment directory:** Isi dengan nama Conda Environment yang Anda buat sendiri (misalnya "tensor"), kecuali jika menggunakan lingkungan lain.
    *   **Software Modules:** Jika Anda memilih partisi "tilla" dan membutuhkan modul CUDA, masukkan nama modul tersebut; jika tidak, biarkan kosong.
    *   **Environment setup & Additional Jupyter arguments:** Isi hanya jika ada instruksi khusus, otherwise biarkan kosong.
    *   **Email address:** Isi alamat email jika ingin menerima notifikasi status mulai dan selesai.
5.  **Mulai Sesi:** Klik tombol "Launch" untuk memulai sesi Jupyter. Anda akan diarahkan ke halaman "My Interactive Sessions".
6.  **Koneksi:** Tunggu hingga status sesi berubah menjadi "Running", kemudian klik tombol "Connect to Jupyter". Sistem akan mulai menghitung Core Hour saat koneksi dilakukan.
7.  **Penggunaan Kernel:** Gunakan kernel "Python 3 (ipykernel)" untuk menjalankan sesi Notebook Anda.

Sesi Jupyter akan terus berjalan selama waktu sesi masih tersedia, bahkan jika koneksi internet terputus atau Anda keluar dari web EFIRO. Untuk menghentikan sesi secara manual, klik tombol "Cancel".

[Q65/69] Apa saja pilihan cara menjalankan komputasi Python dengan conda env di ALELEON?
------------------------------------------------------------
Berdasarkan dokumen referensi, terdapat dua pilihan utama untuk menjalankan komputasi Python dengan conda environment di ALELEON, yaitu Batch Job dan Sesi Jupyter.

1. **Batch Job**: Digunakan untuk menjalankan script Python siap jalan yang tidak membutuhkan interaksi user secara langsung.
   - Job dijalankan oleh Slurm dan user menunggu hingga selesai.
   - File notebook (.ipynb) harus dikonversi terlebih dahulu menjadi file .py.
   - Proses ini berjalan pada Compute node yang terdiri dari partisi **epyc** dan partisi **ampere**.
   - Job akan berhenti jika program selesai, menyentuh limit waktu yang ditentukan, atau menjumpai error prematur.
   - Koneksi user tidak mempengaruhi jalannya batch job; job tetap berjalan meskipun koneksi terputus atau user keluar (log out).

2. **Sesi Jupyter**: Digunakan untuk menjalankan sesi interaktif Jupyter Lab atau Notebook.
   - Sesi ini berjalan pada Interactive node yang terdiri dari partisi **torti** dan partisi **tilla**.
   - Sesi akan berhenti ketika waktu sesi habis, dihentikan manual oleh user, atau menjumpai error prematur.
   - Jika user keluar saat ada cell berjalan dan kembali masuk, proses komputasi mungkin tidak menampilkan status jalannya secara visual, namun koneksi user tidak menghentikan sesi tersebut.
   - User dapat mengecek status cell berikutnya untuk melihat apakah komputasi belum, sedang, atau sudah selesai berjalan.

[Q66/69] Apa perbedaan antara menjalankan batch job via Job Composer EWS dan via terminal Slurm?
------------------------------------------------------------
Perbedaan utama antara menjalankan batch job via Job Composer EWS dan via terminal Slurm terletak pada antarmuka pengguna, metode pembuatan script, serta cara penjadwalan dan pemantauan yang digunakan.

Berikut adalah rincian perbedaannya berdasarkan dokumen referensi:

1. **Antarmuka dan Metode Login**
   - **Job Composer EWS**: User login ke [EWS] atau [EFIRO Web Service], lalu membuka menu Jobs > Job Composer untuk membuat ruang job secara grafis (langkah E1/E2 pada BoltzTraP2/ACPYPE atau langkah 1a/b pada LAMMPS/GMX-MMPBSA).
   - **Terminal Slurm**: User harus membuka terminal Linux melalui opsi [Login SSH] atau menu Clusters > Aleleon Shell Access di [EFIRO Web Service] (Langkah 1 pada GMX-MMPBSA Section 2B).

2. **Persiapan File dan Script**
   - **Job Composer EWS**: User menyiapkan file komputasi via menu [Edit Files] atau [Aplikasi SFTP] dengan batas upload 100MB per file. Script submit dibuat dengan mengklik tombol [Open Editor] di dalam antarmuka web, mengikuti petunjuk NOTES di dalamnya, lalu menyimpannya (Langkah E4-E6 pada BoltzTraP2 atau Langkah 3-4 pada LAMMPS).
   - **Terminal Slurm**: User menyiapkan file di direktori HOME menggunakan [Aplikasi SFTP]. User membuat file script manual dengan format `.sh` (contoh `submit.sh`) yang berisi header `#!/bin/bash -l` dan konfigurasi SBATCH secara teks (Langkah 3 pada GMX-MMPBSA Section 2B).

3. **Cara Menjalankan Job**
   - **Job Composer EWS**: Job dijalankan dengan mengklik tombol [Submit] di antarmuka web (Langkah E7 pada BoltzTraP2 atau Langkah 5 pada LAMMPS).
   - **Terminal Slurm**: Job dijalankan dengan mengetik perintah `sbatch [nama-submit-script]` di terminal (Langkah 5 pada GMX-MMPBSA Section 2B).

4. **Pemantauan Status**
   - **Job Composer EWS**: User memantau status dengan melakukan refresh halaman Job Composer untuk melihat daftar status seperti Not Submitted, Queue, Running, Completed, atau Failed. Output dapat dilihat di kolom Folder Contents (result-jobID.txt dan error-jobID.txt) (Langkah E8 pada BoltzTraP2).
   - **Terminal Slurm**: User memantau status menggunakan perintah `squeue -ul $USER`. Jika tidak ada di squeue, artinya job selesai. Output terminal dapat dilihat menggunakan perintah `ls`, `cat`, `more`, atau `tail` (Langkah 6 pada GMX-MMPBSA Section 2B).

5. **Penghentian Job**
   - **Job Composer EWS**: User dapat menghentikan job dengan klik tombol [Stop] di antarmuka web (Langkah E8 pada BoltzTraP2).
   - **Terminal Slurm**: User menghentikan job dengan menjalankan perintah `scancel [job-ID]` di terminal (Langkah 6 pada GMX-MMPBSA Section 2B).

[Q67/69] Bagaimana langkah lengkap membuat conda env baru dan modul pyload dari awal?

Berikut adalah langkah lengkap untuk membuat conda environment baru dan modul pyload di ALELEON berdasarkan panduan resmi:

5. Selanjutnya, buat modul pyload untuk aktivasi otomatis di sesi berikutnya dengan menjalankan perintah `(env) $ pyl mod`, lalu pilih menu nomor 1 dan ikuti alur script yang muncul.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah panduan lengkap untuk mengisi formulir Jupyter di EWS (EFIRO Web Service) berdasarkan dokumen referensi:

1.  Login ke **EFIRO Web Service** dan buka menu **Interactive Apps > Jupyter**.
2.  Isi kolom **Number of CPU threads** sesuai jumlah core CPU yang dibutuhkan.
3.  Pada opsi **Use GPU for the job**, pilih **no** jika tanpa GPU atau **yes** untuk menggunakan 1x RTX 3090.
4.  Isi kolom **Number of hours** sebagai reservasi waktu sesi Jupyter, yang ditentukan oleh pilihan jumlah CPU, GPU, dan sisa Core Hour Anda.
5.  Pada bagian **Choose package to run**, pilih opsi **User provided**.
6.  Pada bagian **Choose how to launch Jupyter**, pilih **Anaconda / Mamba**.
7.  Pada bagian **Choose Anaconda/Mamba version & User provided Conda/Mamba environment**, masukkan modul Lmod conda dan nama conda env user yang telah dibuat.
    *   Untuk membantu pengisian, Anda dapat membuka menu **Clusters > Aleleon Shell Access**, lalu jalankan perintah `$ pyl info [nama-modul]` untuk melihat info modul pyload user.
8.  Klik tombol **[Launch]** untuk memulai sesi.

Setelah klik Launch, Anda akan diarahkan ke halaman **My Interactive Sessions**. Tunggu hingga status berubah menjadi **Running**, kemudian klik **[Connect to Jupyter]** untuk mulai menggunakan sesi tersebut.
