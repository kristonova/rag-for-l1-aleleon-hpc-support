[Q3/69] Saya ingin pakai partisi GPU. GPU jenis apa yang terpasang di partisi ampere?
------------------------------------------------------------
Partisi `ampere` pada ALELEON Supercomputer dilengkapi dengan 2 unit GPU NVIDIA RTX 3090 berkapasitas 24GB yang saling terhubung melalui NVLink.

Berikut adalah spesifikasi lengkap GPU di partisi tersebut:
1. Model: NVIDIA RTX 3090
2. Kapasitas Memori: 24GB per kartu
3. Jumlah Kartu: 2x (dua) per node
4. Arsitektur: NVIDIA Ampere
5. Fitur Koneksi: Mendukung NVLink untuk paralelisasi multi-GPU

[Q12/69] Bagaimana cara membatalkan/menghentikan job yang berstatus PENDING di terminal?
------------------------------------------------------------
Untuk menghentikan atau membatalkan job yang sedang berjalan (termasuk status PENDING), Anda dapat menjalankan perintah `scancel` diikuti dengan ID job tersebut di terminal.

1.  Pertama, pastikan Anda mengetahui Job ID dari job yang ingin dihentikan dengan melihat hasil perintah `squeue -ul $USER`.
2.  Jalankan perintah berikut untuk membatalkan job:
    `$ scancel [job-ID]`
3.  Sebagai contoh, jika Anda ingin membatalkan job dengan ID 231, ketik perintah:
    `$ scancel 231`

[Q21/69] Saya ingin buka sesi interaktif JupyterLab menggunakan GPU. Apa bedanya partisi torti dan tilla, dan mana yang harus saya pilih?
------------------------------------------------------------
Untuk menjalankan sesi interaktif JupyterLab dengan GPU, Anda harus memilih partisi **tilla**, sedangkan partisi **torti** hanya menyediakan node CPU tanpa dukungan GPU.

Berikut adalah perbedaan spesifikasi dan panduan pemilihannya berdasarkan dokumen:

1.  **Partisi torti (Interactive Node CPU)**
    *   Digunakan untuk komputasi tanpa GPU.
    *   Nodes: 2 buah.

    *   Digunakan untuk komputasi yang memerlukan akselerasi GPU.
    *   Nodes: 2 buah.
    *   CPU per node: AMD Threadripper 3990X, 32 v-core, 2.9Ghz.
    *   RAM per node: 55GB efektif.
    *   GPU per node: 1x NVIDIA RTX 3090 24GB dengan arsitektur NVIDIA Ampere.
    *   Catatan penting: Partisi ini tidak mendukung CUDA versi 11 kebawah.

**Langkah Pemilihan:**
Saat mengisi formulir di menu **Apps > Jupyter** atau **Interactive Apps > Jupyter**:
*   Pada kolom **Partition**, pilih **tilla**.
*   Pada kolom **Number of GPUs** (apabila pilih partition tilla), isi angka **1**.

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Untuk menjalankan file simulasi.ipynb Anda sebagai batch job di Job Composer EWS menggunakan Conda Environment User, ikuti langkah-langkah berikut sesuai dokumen:

1. Login ke [EFIRO Web Service], buka menu Jobs > Job Composer, lalu buat ruang job dengan memilih menu New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" untuk berjalan di compute node CPU atau "Conda Environment User (GPU)" jika membutuhkan akselerasi GPU, kemudian isi kolom Job Name (bebas) dan klik tombol [Create New Job].
3. Siapkan file komputasi melalui menu [Edit Files] yang memungkinkan upload, download, manajemen, dan edit file dengan batas maksimal 100MB per file; jika ukuran file melebihi batas, gunakan [Aplikasi SFTP] ke direktori Job Composer.
4. Karena batch job hanya menjalankan file Python (.py), lakukan konversi file Notebook menjadi format Python dengan membuka [Open Terminal] dan jalankan perintah: $ jupytext --to py simulasi.ipynb, yang akan menghasilkan file simulasi.py.
5. Lakukan pengecekan kelengkapan environment dengan membuka [Open Terminal], mengaktifkan modul pyload user yang digunakan dengan perintah $ pyl load [nama-modul], lalu jalankan perintah pengecekan $ pycheck simulasi.py.
6. Perhatikan output dari perintah pycheck tersebut: jika muncul "READY TO GO!" berarti file siap dijalankan, sedangkan jika muncul "MISSING PACKAGE!", instal package yang kurang sesuai panduan output dengan perintah contoh seperti $ pip install --user pyusb.
7. Isi kolom submit script untuk menulis alur jalannya job dengan cara klik [Open Editor], ikuti petunjuk NOTES dan alur script di dalamnya, lalu klik [Save] untuk menyimpan script.
8. Jalankan job dengan mengklik tombol [Submit].
9. Pantau status dan output job dengan meremfresh halaman Job Composer; status dapat berupa Not Submitted, Queue, Running, Completed, atau Failed.
10. Jika ingin memantau output, lihat di kolom Folder Contents yang akan menampilkan hasil result-jobID.txt untuk output terminal dan error-jobID.txt untuk log status.
11. Untuk mendownload file output, gunakan opsi Menu [Edit Files] atau arahkan [Aplikasi SFTP] ke direktori Job Composer.

[Q30/69] Apa bedanya Effective Core Hour dengan Actual Core Hour di dalam sistem ALELEON?
------------------------------------------------------------
Berikut adalah perbedaan antara Effective Core Hour dan Actual Core Hour berdasarkan sistem ALELEON:

1. **Effective Core Hour Remaining** adalah sisa kuota core hour efektif yang menjadi dasar Slurm ALELEON untuk menjalankan job, di mana nilainya berubah tergantung status job:
   - Ketika tidak ada job berjalan, Effective Core Hour Remaining sama dengan sisa kuota Actual Core Hour.
   - Ketika ada job berjalan atau tertahan, Effective Core Hour Remaining dihitung dari sisa kuota Total Core Hour (yang mencakup Actual Core Hour ditambah Booked Core Hour).

2. **Actual Core Hour Usage** adalah penggunaan core hour sesungguhnya secara realtime yang menjadi basis perhitungan biaya penggunaan ALELEON, dan nilainya hanya bertambah saat job benar-benar berjalan sesuai durasi yang digunakan.

[Q32/69] (Troubleshooting) Saya submit job GROMACS tapi selalu gagal dengan pesan error Invalid syntax. Di script saya menulis #SBATCH --mem= 64 GB. Apa yang salah dari tulisan saya?
------------------------------------------------------------
Masalah pada penulisan perintah `#SBATCH --mem= 64 GB` terletak pada adanya spasi yang tidak seharusnya di antara tanda sama dengan (`=`) dan nilai, serta adanya spasi di dalam satuan memori.

Berdasarkan dokumen referensi, format penulisan SBATCH yang benar **tidak boleh** memiliki spasi di parameter, tanda sama dengan, maupun nilai. Berikut adalah perbandingan kesalahan dan kebenaran sesuai panduan:

1.  **Penulisan Salah (seperti yang Anda lakukan):**
    *   `#SBATCH --mem= 64 GB` (Spasi setelah `=` dan sebelum `GB`)
    *   `#SBATCH --mem= 64GB` (Spasi setelah `=`)
    *   `#SBATCH --time =01:00:00` (Spasi sebelum `=`)

2.  **Penulisan Benar:**
    *   `#SBATCH --mem=64GB` (Tanpa spasi sama sekali antara nama variabel, tanda sama dengan, dan nilai)
    *   Contoh lain yang benar: `#SBATCH --ntasks=32` atau `#SBATCH --time=01:00:00`.

Anda perlu mengubah baris tersebut menjadi `#SBATCH --mem=64GB` tanpa spasi agar sintaksnya valid dan job dapat berjalan.

[Q35/69] Saya mau menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array. Setiap simulasi butuh 4 core CPU dan 8GB RAM. Di script, apakah saya harus menulis --cpus-per-task=40 atau --cpus-per-task=4?
------------------------------------------------------------
Untuk menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan Slurm Array dengan setiap simulasi membutuhkan 4 core CPU, Anda harus menulis `#SBATCH --cpus-per-task=4` pada script submit.

Berdasarkan dokumen referensi, alokasi `--cpus-per-task` digunakan oleh **setiap** job array secara individual, bukan total untuk seluruh array. Berikut adalah rincian logika dan perintah yang benar sesuai dokumen:

1.  **Penulisan Perintah**: Tulis `#SBATCH --cpus-per-task=4` karena nilai ini merepresentasikan jumlah thread CPU per satu tugas (per simulasi).
2.  **Pengaturan Array**: Pastikan Anda juga menetapkan parameter array untuk jumlah simulasi dengan perintah `#SBATCH --array=1-10`.
3.  **Perhitungan Total Alokasi**: Sistem akan menghitung total alokasi secara otomatis berdasarkan rumus: `Total CPU = cpus-per-task * jumlah_array`. Dalam kasus Anda, total alokasi CPU akan menjadi $4 \times 10 = 40$ core.
4.  **Contoh Script Lengkap**: Sesuai template "FLACS-CFD Array" di dokumen, struktur bagian spesifikasi jobnya adalah sebagai berikut:

```bash
# Jumlah job array, contoh: 1-8
#SBATCH --array=1-10

# Jumlah core thread CPU, minimal 2
# Total CPU = cpus-per-task * array
#SBATCH --cpus-per-task=4

# Jumlah memori RAM (satuan GB)
# Total memori = mem * array
#SBATCH --mem=8GB
```

Jika Anda menulis `--cpus-per-task=40`, sistem akan mengalokasikan 40 core CPU untuk **setiap** dari 10 simulasi tersebut, sehingga total penggunaan CPU akan menjadi 400 core, yang jauh melebihi kebutuhan Anda.

[Q39/69] Kenapa saat saya meminta alokasi #SBATCH --ntasks=7, sistem Slurm ALELEON akan membulatkannya menjadi 8 dan saya ditagih biaya untuk 8 core?
------------------------------------------------------------
Sistem Slurm ALELEON membulatkan alokasi core CPU ganjil menjadi genap ke atas demi optimasi performa karena ALELEON mengaktifkan SMT (Simultaneous Multithreading) di mana setiap core memiliki 2 thread yang tidak boleh dipakai oleh job berbeda.

Berdasarkan aturan, jumlah alokasi CPU ganjil akan otomatis dibulatkan genap ke atas, contohnya jumlah 7 akan berubah menjadi 8 dan jumlah 23 menjadi 24.
Oleh karena itu, ketika Anda meminta `#SBATCH --ntasks=7`, sistem mengubahnya menjadi 8 sesuai dengan ketentuan alokasi minimum 2 dan genap tersebut.

[Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Berikut adalah langkah-langkah lengkap untuk membuat Conda Environment di ALELEON berdasarkan dokumentasi resmi:

1. Buka terminal ALELEON melalui opsi Login SSH atau menu Clusters > Aleleon Shell Access pada EFIRO Web Service.
2. Aktifkan modul Lmod conda yang sesuai dengan kebutuhan Anda, misalnya versi terbaru Anaconda 3 2025.06-1 dengan perintah `$ ml Anaconda3/2025.06-1`, atau gunakan versi stabil lainnya seperti Anaconda3 2024.02-1 dengan perintah `$ ml Anaconda3/2024.02-1`.
3. Buat environment kosong menggunakan perintah `conda create -n [nama-env] python=[versi-python]` (contoh: `$ conda create -n skripsi python=3.12`) atau jika menggunakan Mamba, gunakan `$ mamba create -n [nama-env] python=[versi-python]`.
4. Setelah environment dibuat, aktifkan environment tersebut dengan perintah `$ source activate [nama-env]` dan selanjutnya buat modul pyload untuk kemudahan aktivasi di masa depan dengan menjalankan `$ pyl mod` lalu ikuti alur script yang muncul.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Untuk menjalankan sesi Jupyter menggunakan Conda Environment (conda env) sendiri di ALELEON, Anda perlu mengikuti langkah-langkah berikut melalui EFIRO Web Service:

1. Login ke web EFIRO ALELEON Supercomputer dan klik aplikasi **Jupyter** pada homepage atau pilih menu **Apps > Jupyter**.
2. Isi formulir untuk memulai sesi dengan memilih **Partition** sesuai kebutuhan (gunakan `torti` untuk interactive node CPU atau `tilla` untuk interactive node GPU).
3. Pada bagian **Choose how to launch Jupyter**, pilih opsi **Anaconda/Mamba**.
4. Pilih versi **Anaconda 3 2024.02-1** pada kolom **Choose Anaconda/Mamba version**.
5. Pada kolom **Conda/Mamba environment directory**, isi nama conda environment yang telah Anda buat (misalnya `tensor`), kecuali jika Anda ingin menggunakan nama lain yang sudah dibuat sebelumnya.
6. Pastikan Anda telah menginstal package yang diperlukan di dalam conda environment tersebut sebelum memulai sesi untuk efisiensi waktu dan Core Hour.
7. Klik tombol **Launch** untuk memulai sesi; sistem akan mengarahkan Anda ke halaman **My Interactive Sessions** hingga sesi siap diakses.

Setelah status berubah menjadi **Running**, klik tombol **Connect to Jupyter** untuk mulai bekerja, dan pastikan menggunakan kernel **Python 3 (ipykernel)** di dalam notebook Anda.

[Q67/69] Bagaimana langkah lengkap membuat conda env baru dan modul pyload dari awal?
------------------------------------------------------------
Berikut adalah langkah lengkap untuk membuat conda environment baru dan modul pyload dari awal berdasarkan dokumen referensi:

1. Buka terminal ALELEON melalui opsi Login SSH atau menu EFIRO Web Service pada Clusters > Aleleon Shell Access.
2. Pilih dan aktifkan modul Lmod conda yang sesuai; tim admin menyarankan versi terbaru Anaconda3/2025.06-1 (Python 3.13.4) atau Mamba/23.11.0-0 (Python 3.10.13), namun disarankan menghindari Anaconda3/2024.06-1 karena bug.
   - Perintah aktivasi contoh: `$ ml Anaconda3/2025.06-1`
3. Buat conda environment dengan format perintah berikut, pastikan versi Python didefinisikan untuk isolasi site-packages:
   - Menggunakan Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
   - Contoh perintah: `$ conda create -n skripsi python=3.12`
   - Jika menggunakan Mamba: `$ mamba create -n [nama-env] python=[versi-python]`
4. Aktifkan conda environment yang baru dibuat menggunakan perintah source:
   - Perintah: `$ source activate [nama-env]`
5. Buat modul pyload untuk aktivasi conda environment berikutnya dengan menjalankan perintah `pyl mod`:
   - Perintah: `(env) $ pyl mod`
   - Ikuti alur script yang muncul di terminal dengan memilih menu no.1.

[Q68/69] Apa saja status job di squeue dan artinya masing-masing?
------------------------------------------------------------
Berikut adalah daftar status job (STATE) yang ditampilkan pada kolom ST atau squeue beserta penjelasannya:

1. R (RUN): Menunjukkan bahwa job sedang berjalan.
2. PD (PENDING): Menunjukkan bahwa job tertahan, pengguna perlu melihat NODELIST(REASON) untuk mengetahui alasannya.
3. CG (COMPLETING): Menunjukkan bahwa job telah selesai dan sedang dalam proses clean-up.
4. CA (CANCELED): Menunjukkan bahwa job dibatalkan oleh user.
5. PR (PREEMPETED): Menunjukkan bahwa job dibatalkan oleh admin, di mana alasan pemberitahuan dikabarkan melalui email.
6. S (SUSPENDED): Menunjukkan bahwa job ditahan oleh admin, di mana alasan pemberitahuan dikabarkan melalui email.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah langkah-langkah mengisi formulir untuk memulai sesi Jupyter di EWS sesuai panduan:

1.  Klik aplikasi **Jupyter** pada homepage EFIRO atau masuk melalui menu **Apps > Jupyter**.
2.  Pada bagian **Choose Jupyter version**, pilih preferensi Anda antara **Jupyter Lab** atau **Notebook**.
3.  Pada bagian **Partition**, pilih **torti** untuk interactive node CPU atau **tilla** untuk interactive node GPU.
4.  Isi **Number of CPU thread(s)** dengan angka dalam rentang **2 - 16**.
5.  Isi **Amount of memory/RAM (GB)** dengan angka dalam rentang **1 - 55**.
6.  Jika memilih partition **tilla**, isi **Number of GPUs** dengan angka **1**.
7.  Isi **Number of hours** dengan waktu maksimal sesi dalam rentang **1 - 24**.
8.  Pada bagian **Choose how to launch Jupyter**, pilih opsi **Anaconda/Mamba**.
9.  Pada bagian **Choose Anaconda/Mamba version**, pilih versi **Anaconda 3 2024.02-1**.
10. Pada bagian **Conda/Mamba environment directory**, isi nama conda env yang dibuat user (contoh: **tensor**), kecuali jika menggunakan conda env lain maka isi dengan nama conda env tersebut.
11. Pada bagian **Software Modules**, isi nama modul CUDA hanya jika diperlukan dan tidak menggunakan conda env 'tensor' (jika butuh, isi nama modul CUDA).
12. Isi kolom **Environment setup** dan **Additional Jupyter arguments** hanya jika ada variabel lingkungan atau argumen tambahan yang diperlukan.
13. Isi **Email address** jika ingin menerima notifikasi status mulai dan selesainya sesi Jupyter.
14. Klik tombol **Launch** untuk memulai sesi Jupyter.
