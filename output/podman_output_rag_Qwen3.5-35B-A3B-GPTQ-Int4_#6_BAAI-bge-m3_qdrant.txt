[Q9/69] Jika saya pakai aplikasi SFTP seperti FileZilla, apakah ada limit ukuran file yang bisa diupload?
------------------------------------------------------------
Ya, ada batasan ukuran file untuk upload melalui aplikasi SFTP seperti FileZilla. Berdasarkan dokumen referensi, limit upload adalah **100MB per file**. Jika ukuran file Anda melebihi 100MB, Anda disarankan untuk menggunakan aplikasi SFTP (karena opsi via EFIRO Web Service memiliki batas yang sama), namun jika dokumen merujuk pada transfer via web service, maka untuk file lebih besar dari 100MB harus menggunakan aplikasi SFTP.

Namun, perlu diperhatikan bahwa dokumen juga menyebutkan bahwa **tidak ada limit ukuran upload/download** secara umum untuk akses SFTP langsung ke HOME user, kecuali jika merujuk pada transfer via Job Composer atau Web Service yang memiliki batas 100MB per file. Untuk penggunaan SFTP standar ke direktori HOME, tidak disebutkan adanya batas maksimal ukuran file, sehingga asumsinya tidak ada limit.

[Q12/69] Bagaimana cara membatalkan/menghentikan job yang berstatus PENDING di terminal?
------------------------------------------------------------
Untuk membatalkan atau menghentikan job yang berstatus PENDING di terminal, Anda dapat menggunakan perintah `scancel` diikuti dengan ID job tersebut. Berikut adalah langkah-langkahnya:

1. Lihat status dan Job ID job Anda dengan menjalankan perintah `$ squeue -ul $USER`.
2. Jalankan perintah pembatalan dengan format `$ scancel [job-ID]`, contohnya untuk membatalkan job ID 231 adalah `$ scancel 231`.
3. Setelah dibatalkan, status job akan berubah menjadi CA (CANCELED) yang berarti job dibatalkan oleh user.

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Untuk menjalankan file simulasi.ipynb sebagai batch job di Job Composer EWS menggunakan Conda Environment User, berikut adalah urutan langkah-langkah yang harus Anda lakukan:

1. Login ke EFIRO Web Service, buka menu Jobs > Job Composer, lalu buat ruang job dengan memilih New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" untuk berjalan di compute node CPU atau "Conda Environment User (GPU)" jika membutuhkan akselerasi GPU di compute node GPU, kemudian isi Job Name dan klik [Create New Job].
3. Siapkan file komputasi via menu [Edit Files] dengan upload file simulasi.ipynb (limit upload 100MB per file; bila lebih gunakan Aplikasi SFTP).
4. Karena batch job hanya menjalankan file Python (.py), buka [Open Terminal] dan lakukan konversi file Notebook ke Python dengan perintah:
   $ jupytext --to py simulasi.ipynb
   Perintah ini akan menghasilkan file Python simulasi.py.
5. Lakukan pengecekan kelengkapan environment dengan membuka [Open Terminal], mengaktifkan modul pyload user yang akan digunakan ($ pyl load [nama-modul]), lalu jalankan pengecekan pada file Python yang akan dijalankan:
   $ pycheck simulasi.py
6. Perhatikan output dari perintah pycheck tersebut:
   - Jika muncul READY TO GO!, maka file siap dijalankan.
   - Jika muncul MISSING PACKAGE!, instal package yang kurang sesuai panduan output, contoh:
     $ pip install --user [nama-package]
7. Isi kolom submit script untuk menulis alur jalannya job dengan klik [Open Editor], ikuti petunjuk NOTES dan alur script di dalamnya, lalu klik [Save] untuk menyimpan script.
8. Jalankan job dengan klik tombol [Submit].
9. Pantau status dan output job dengan refresh halaman Job Composer untuk update status, lihat output di kolom Folder Contents (result-jobID.txt untuk output terminal dan error-jobID.txt untuk log status), atau pantau utilitas CPU dan memori melalui Monitoring > Grafana.

[Q30/69] Apa bedanya Effective Core Hour dengan Actual Core Hour di dalam sistem ALELEON?
------------------------------------------------------------
Berikut adalah perbedaan antara Effective Core Hour dan Actual Core Hour berdasarkan sistem ALELEON:

1. **Effective Core Hour Remaining** adalah sisa kuota core hour efektif yang menjadi dasar Slurm menjalankan job, di mana nilainya berubah tergantung status job:
   - Jika tidak ada job berjalan, nilainya sama dengan sisa kuota Actual Core Hour.
   - Jika ada job berjalan, nilainya sama dengan sisa kuota Total Core Hour (yang mencakup Booked Core Hour).

2. **Actual Core Hour Usage** adalah penggunaan core hour sesungguhnya secara *realtime* yang dihitung dari waktu job benar-benar berjalan, dan menjadi basis perhitungan biaya penggunaan ALELEON.

Secara ringkas, *Effective Core Hour* mencerminkan sisa kuota yang tersedia untuk alokasi (termasuk yang sudah dipesan), sedangkan *Actual Core Hour* hanya menghitung konsumsi sumber daya yang sudah benar-benar dipakai oleh job yang sedang aktif.

[Q33/69] Saya submit 3 batch job berturut-turut. Job 1 pakai 64 CPU. Job 2 pakai 32 CPU. Kenapa saat saya submit Job 3 yang butuh 64 CPU, statusnya malah PENDING dengan tulisan QOSMaxCpuPerUserLimit, padahal node epyc masih banyak yang kosong?
------------------------------------------------------------
Status PENDING dengan alasan `QOSMaxCpuPerUserLimit` pada Job 3 terjadi karena alokasi CPU yang Anda minta melebihi batas limit komputasi (Fair Usage Limit/LFU) per pengguna, bukan karena kekosongan node fisik.

Berdasarkan dokumen referensi, berikut adalah penjelasan dan langkah-langkah yang harus Anda lakukan:

1.  **Penyebab Masalah**
    Sistem Slurm menahan job Anda karena total alokasi CPU melebihi limit LFU akun perseorangan Anda. Meskipun node epyc kosong secara fisik, sistem membatasi jumlah CPU yang bisa dialokasikan sekaligus untuk satu user agar adil bagi semua pengguna.
    *   Batas maksimal alokasi CPU untuk **Akun Perseorangan** adalah **128 core**.
    *   Jika Job 1 (64 CPU) dan Job 2 (32 CPU) sedang berjalan atau antri, total alokasinya sudah mencapai 96 CPU. Jika Job 3 meminta tambahan 64 CPU, totalnya akan menjadi 160 CPU, yang jelas melebihi batas 128 CPU tersebut.

2.  **Langkah Cek Awal**
    Jalankan tool `sausage` menu `sfair` untuk melihat besar limit komputasi (LFU) dan penggunaan saat ini di kolom "Fair Limit" serta "Current Usage".
    Perintah yang digunakan adalah:
    ```bash
    sausage sfair
    ```

3.  **Analisis Padanan SBATCH vs Fair Limit**
    Bandingkan parameter SBATCH Job 3 Anda dengan limit yang tertera di `sausage sfair`. Perhatikan bahwa padanan untuk CPU adalah hasil perkalian `ntasks` dikali `cpus-per-task`.
    *   Jika `ntasks * cpus-per-task` (CPU) melebihi nilai di kolom "Fair Limit", maka job akan tertahan.

4.  **Solusi yang Tersedia**
    Jika cek menunjukkan ada yang lebih dari limit, Anda memiliki dua opsi sesuai prosedur dokumentasi:
    
    *   **Opsi 1: Batalkan dan Revisi Job**
        Batalkan job yang berlebih (Job 3), ubah jumlah alokasi CPU di script SBATCH agar sesuai dengan sisa limit yang tersedia, lalu jalankan ulang.
        
        Cara membatalkan job ID (misalnya ID 231) melalui terminal:
        ```bash
        scancel 231
        ```
        
        Atau melalui EFIRO Web Service:
        1. Buka menu **Apps > Active Jobs**.
        2. Ubah opsi **All Jobs** ke **Your Jobs**.
        3. Cari job yang akan dibatalkan.
        4. Pada kolom Action, klik icon **Delete Job**.
        
        Setelah dibatalkan, edit file submit script Anda untuk mengurangi nilai `--ntasks` atau `--cpus-per-task` sehingga total CPU tidak melebihi batas 128 core, kemudian jalankan kembali dengan perintah `sbatch [nama-submit-script]`.

    *   **Opsi 2: Ajukan Pembukaan Limitasi Sementara**
        Ajukan pembukaan limitasi sementara kepada tim support admin. Tim admin akan memutuskan permintaan Anda menyesuaikan ketersediaan sumber daya komputasi dan antrian job pengguna lain di ALELEON Supercomputer.

5.  **Catatan Penting**
    Pastikan Anda juga memeriksa apakah ada job lain milik Anda yang sedang berjalan (RUNNING) atau antri (PENDING) yang memakan alokasi CPU, karena akumulasi dari semua job aktiflah yang dibandingkan dengan limit LFU.

[Q34/69] (Troubleshooting) Job saya berstatus PD dengan alasan AssocMaxWallDurationPerJobLimit. Di script saya menulis #SBATCH --time=4-00:00:00. Akun saya adalah akun perseorangan biasa. Mengapa tertahan?
------------------------------------------------------------
Job Anda tertahan dengan status **PENDING** dan alasan **AssocMaxWallDurationPerJobLimit** karena durasi waktu yang Anda ajukan melebihi batas maksimal walltime yang diperbolehkan untuk akun perseorangan.

Berdasarkan dokumen referensi, berikut adalah rincian penyebabnya:
1.  **Batas Maksimal Walltime:** Untuk akun perseorangan (Akun Perseorangan), alokasi komputasi maksimal hanya dibatasi hingga **3 hari (72 jam)**.
2.  **Analisis Script Anda:** Anda menulis `#SBATCH --time=4-00:00:00` yang setara dengan **4 hari**. Karena 4 hari lebih besar dari batas maksimal 3 hari, sistem Slurm akan menahan job tersebut (status PENDING) sesuai aturan Fair Usage Limit (LFU).

Untuk memperbaiki hal ini, Anda perlu mengubah nilai waktu di dalam script submit menjadi maksimal **3 hari (72 jam)** atau format **3-00:00:00**, atau format alternatif **72:00:00**.

[Q35/69] Saya mau menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array. Setiap simulasi butuh 4 core CPU dan 8GB RAM. Di script, apakah saya harus menulis --cpus-per-task=40 atau --cpus-per-task=4?
------------------------------------------------------------
Untuk menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan Slurm Array di mana setiap simulasi membutuhkan 4 core CPU, Anda harus menulis `#SBATCH --cpus-per-task=4` pada script submit.

Berdasarkan dokumen referensi, alokasi `--cpus-per-task` (core thread CPU) digunakan oleh **setiap** array job secara individual, bukan untuk total seluruh job. Berikut adalah rincian perhitungannya sesuai contoh dalam dokumen:
1. Jika Anda mengatur `#SBATCH --array=1-10` untuk 10 simulasi dan `#SBATCH --cpus-per-task=4`, maka total alokasi CPU yang terpakai adalah 10 * 4 = 40 core.
2. Jika Anda menulis `#SBATCH --cpus-per-task=40`, maka total alokasi CPU akan menjadi 10 * 40 = 400 core, yang jauh melebihi kebutuhan Anda.

Demikian pula untuk memori RAM, jika Anda menulis `#SBATCH --mem=8GB`, sistem akan mengalokasikan 8GB untuk setiap simulasi (total 80GB), sehingga perintah tersebut sudah benar untuk kebutuhan per simulasi Anda.

[Q39/69] Kenapa saat saya meminta alokasi #SBATCH --ntasks=7, sistem Slurm ALELEON akan membulatkannya menjadi 8 dan saya ditagih biaya untuk 8 core?
------------------------------------------------------------
Sistem Slurm ALELEON membulatkan alokasi core CPU ganjil menjadi genap ke atas demi optimasi performa karena teknologi SMT (Simultaneous Multi-Threading) yang mengaktifkan 2 thread per core.

Berdasarkan dokumen referensi, berikut adalah penjelasan detailnya:

1.  **Alasan Teknis**: ALELEON mengaktifkan SMT pada semua CPU compute node sehingga berjalan 2 thread per core. Slurm dikonfigurasi agar 2 thread dalam 1 core CPU tidak dipakai oleh 2 job berbeda untuk menjaga optimasi performa.
2.  **Aturan Pembulatan**: Jumlah alokasi CPU ganjil akan otomatis dibulatkan genap ke atas oleh Slurm. Contoh yang tercantum di dokumen menyatakan bahwa jumlah 7 akan menjadi 8, dan 23 menjadi 24.
3.  **Dampak Tagihan**: Karena sistem mengalokasikan sumber daya fisik sebesar 8 core untuk menjalankan job tersebut, maka perhitungan biaya Core Hour (CCH) didasarkan pada alokasi aktual yang diberikan sistem, yaitu 8 core, bukan angka awal yang diminta user.

[Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Untuk membuat Conda Environment di ALELEON, Anda perlu membuka terminal melalui menu **Apps > Aleleon Shell Access** atau menggunakan **Login SSH**, kemudian aktifkan modul Anaconda yang sesuai dengan perintah `$ module load Anaconda3/[versi]`. Setelah itu, buat lingkungan baru dengan menjalankan perintah `$ conda create -n [nama-env] python=[versi-python]` atau menggunakan Mamba dengan `$ mamba create -n [nama-env] python=[versi-python]`.

Berikut adalah langkah-langkah spesifik berdasarkan versi dan panduan yang tersedia:

1. Buka terminal ALELEON dengan opsi Login SSH atau menu Apps > Aleleon Shell Access.
2. Aktifkan modul Lmod conda yang disarankan (misalnya versi terbaru) dengan perintah seperti `$ ml Anaconda3/2025.06-1`, `$ ml Anaconda3/2024.06-1`, atau `$ ml Mamba/23.11.0-0`.
   *Catatan: Hindari penggunaan `Anaconda3 2024.06-1` karena adanya laporan bug.*
3. Buat conda environment kosong dengan format berikut:
   - Untuk Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
   - Untuk Mamba: `$ mamba create -n [nama-env] python=[versi-python]`
   - Contoh pembuatan env bernama "skripsi" dengan Python 3.12 menggunakan Anaconda: `$ conda create -n skripsi python=3.12`.
4. Jika Anda memiliki file YAML resep instalasi package, Anda dapat membuatnya langsung dengan perintah `$ conda create -f [nama-file-yml].yml` atau `$ mamba create -f [nama-file-yml].yml`.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Berikut adalah langkah-langkah untuk menjalankan sesi Jupyter menggunakan Conda Environment sendiri di sistem ALELEON:

1.  **Persiapan dan Instalasi Package**
    Pastikan Anda telah mengaktifkan Conda Environment dan menginstal package yang diperlukan sebelum memulai sesi interaktif agar efisien dan menghindari error.
    *   Buka terminal melalui menu **Apps > Aleleon Shell Access** atau via **Login SSH**.
    *   Muat modul Anaconda dengan perintah: `module load Anaconda3/2024.02-1`.
    *   Aktifkan Conda Environment Anda dengan perintah: `source activate [nama-conda-env]` (ganti `[nama-conda-env]` dengan nama lingkungan Anda, contoh: `tensor`). Jika lupa nama env, gunakan perintah `conda env list`.
    *   Instal package Jupyter jika belum ada dengan perintah: `pip install jupyter jupyterlab`.

2.  **Login ke Web EFIRO**
    Masuk ke halaman web **EFIRO Web Service** ALELEON Supercomputer.

3.  **Isi Formulir Sesi Jupyter**
    Klik aplikasi **Jupyter** di homepage atau buka menu **Apps > Jupyter**, lalu isi formulir berikut sesuai kebutuhan:
    *   **Choose Jupyter version**: Pilih preferensi Anda antara **Jupyter Lab** atau **Notebook**.
    *   **Partition**: Pilih partisi sesuai kebutuhan komputasi:
        *   **torti**: Untuk interactive node CPU.
        *   **tilla**: Untuk interactive node GPU (perlu alokasi GPU).
    *   **Number of CPU thread(s)**: Masukkan jumlah core CPU (range 2 - 16 untuk akun uji coba, atau 1 - 32 untuk akun perseorangan).
    *   **Amount of memory/RAM (GB)**: Masukkan alokasi RAM (range 1 - 55 untuk akun uji coba, atau 1 - 64 untuk akun perseorangan).
    *   **Number of GPUs**: Isi angka **1** jika memilih partisi **tilla**.
    *   **Number of hours**: Tentukan waktu maksimal sesi (range 1 - 24 jam untuk akun uji coba, atau 1 - 72 jam untuk akun perseorangan).
    *   **Choose how to launch Jupyter**: Pilih **Anaconda/Mamba**.
    *   **Choose Anaconda/Mamba version**: Pilih **Anaconda 3 2024.02-1**.
    *   **Conda/Mamba environment directory**: Isi dengan nama Conda Environment yang Anda buat (contoh: **tensor**), atau nama modul pyload user jika sudah dikonfigurasi.
    *   **Software Modules**: Kosongkan kecuali Anda memerlukan modul tambahan seperti CUDA (untuk partisi tilla), namun perhatikan bahwa Conda env 'tensor' tidak perlu memuat modul CUDA secara manual.
    *   **Environment setup** & **Additional Jupyter arguments**: Kosongkan jika tidak ada instruksi khusus.
    *   **Email address**: Isi jika ingin menerima notifikasi status mulai dan selesai.

4.  **Mulai Sesi**
    Klik tombol **Launch** untuk memulai sesi. Anda akan diarahkan ke halaman **My Interactive Sessions**. Tunggu hingga status berubah menjadi **Running**.

5.  **Koneksi dan Penggunaan**
    Ketika status sudah **Running**, klik tombol **Connect to Jupyter**. Sistem akan mulai menghitung Core Hour. Gunakan kernel **Python 3 (ipykernel)** untuk sesi Notebook Anda. Sesi akan terus berjalan selama waktu sesi masih ada, bahkan jika koneksi terputus atau Anda keluar dari web EFIRO. Untuk menghentikan sesi, klik tombol **Cancel**.

[Q66/69] Apa perbedaan antara menjalankan batch job via Job Composer EWS dan via terminal Slurm?
------------------------------------------------------------
Perbedaan utama antara menjalankan batch job via Job Composer EWS dan via terminal Slurm terletak pada antarmuka pengguna, cara pembuatan script, serta metode pemantauan yang digunakan.

Berikut adalah rincian perbedaannya berdasarkan dokumen referensi:

1.  **Antarmuka dan Cara Login**
    *   **Job Composer EWS:** User melakukan login ke [EWS] atau [EFIRO Web Service], lalu membuka menu Jobs > Job Composer untuk membuat ruang kerja secara grafis.
    *   **Terminal Slurm:** User harus membuka terminal ALELEON melalui opsi [Login SSH] atau menu Clusters > Aleleon Shell Access di [EFIRO Web Service].

2.  **Persiapan File dan Script**
    *   **Job Composer EWS:** User menyiapkan file komputasi melalui menu [Edit Files] (dengan batas upload 100MB per file) atau menggunakan [Aplikasi SFTP]. Untuk submit script, user mengklik tombol [Open Editor] di dalam antarmuka web, mengikuti petunjuk NOTES, lalu menyimpannya sebelum klik [Submit].
    *   **Terminal Slurm:** User menyiapkan file komputasi di direktori HOME menggunakan [Aplikasi SFTP] atau login EWS > Files > Home Directory. User membuat file submit script sendiri dengan format .sh (contoh: `submit.sh`) secara manual di terminal, memastikan header `#!/bin/bash -l` ada, dan mengisi bagian spesifikasi SBATCH sesuai template.

3.  **Cara Menjalankan Job**
    *   **Job Composer EWS:** Job dijalankan dengan mengklik tombol [Submit] yang tersedia di antarmuka web setelah script disimpan.
    *   **Terminal Slurm:** Job dijalankan dengan mengetik perintah shell `sbatch [nama-submit-script]` (contoh: `$ sbatch submit.sh`).

4.  **Pemantauan Status dan Output**
    *   **Job Composer EWS:** User memantau status dengan me-refresh halaman Job Composer. Status ditampilkan sebagai Not Submitted, Queue, Running, Completed, atau Failed. Output dilihat di kolom Folder Contents (file `result-jobID.txt` dan `error-jobID.txt`). Untuk utilitas CPU/Memori, user membuka menu Monitoring > Grafana lalu Jobs > Active Jobs.
    *   **Terminal Slurm:** User memantau status dengan perintah `squeue -ul $USER`. Status ditampilkan dalam kode ST (PD untuk Pending, R untuk Run). Output dilihat langsung di terminal menggunakan perintah seperti `ls`, `cat`, `more`, atau `tail`. Untuk menghentikan job, user menggunakan perintah `scancel [job-ID]`.

5.  **Alat Bantu Tambahan**
    *   **Job Composer EWS:** Menggunakan tool `sausage` di dalam terminal yang dibuka dari Job Composer untuk menghitung rentang alokasi (CPU, memori, waktu) dengan memilih menu `slimit` -> `Batch Job`.
    *   **Terminal Slurm:** Juga menggunakan tool `sausage` di terminal Linux untuk panduan rentang nilai, namun pemantauan lebih terintegrasi langsung dengan perintah baris standar Slurm (`squeue`, `scancel`).

[Q67/69] Bagaimana langkah lengkap membuat conda env baru dan modul pyload dari awal?
------------------------------------------------------------
Berikut adalah langkah lengkap untuk membuat conda environment baru dan modul pyload di ALELEON berdasarkan panduan resmi:

1. Buka terminal ALELEON melalui opsi Login SSH atau masuk ke menu Clusters > Aleleon Shell Access pada EFIRO Web Service.
2. Aktifkan modul Lmod Conda yang diinginkan, misalnya dengan perintah `$ ml Anaconda3/2025.06-1` untuk versi terbaru (hindari versi 2024.06-1 karena ada laporan bug).
3. Buat conda environment baru menggunakan format perintah berikut sesuai kebutuhan package Anda:
   - Untuk Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
   - Atau jika ada file YAML: `$ conda create -f [nama-file-yml].yml`
   - Contoh spesifik: `$ conda create -n skripsi python=3.12`
4. Aktifkan conda environment tersebut dengan perintah `$ source activate [nama-env]`.
5. Setelah aktif, buat modul pyload untuk aktivasi otomatis di sesi berikutnya dengan menjalankan perintah `(env) $ pyl mod`, lalu ikuti alur script yang muncul dengan memilih menu no.1.

[Q68/69] Apa saja status job di squeue dan artinya masing-masing?
------------------------------------------------------------
Berikut adalah daftar status job (ST / STATE) yang ditampilkan oleh perintah `squeue` beserta artinya:

1. PD (PENDING) - Job mengantri dan belum berjalan atau tertahan; jika tertahan dalam waktu lama, disarankan untuk melihat bagian Troubleshooting 1.
2. R (RUN) - Job sedang berjalan saat ini.
3. CG (COMPLETING) - Job telah selesai namun masih dalam proses pembersihan (clean-up).
4. CA (CANCELED) - Job dibatalkan secara manual oleh user.
5. PR (PREEMPETED) - Job dibatalkan oleh tim admin, dengan notifikasi alasan dikirimkan melalui email support.
6. S (SUSPENDED) - Job ditahan sementara oleh tim admin, dengan notifikasi alasan dikirimkan melalui email support.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah panduan lengkap untuk mengisi formulir sesi Jupyter di EWS (EFIRO Web Service) berdasarkan dokumen referensi:

1.  **Login ke EFIRO** dan buka menu **Interactive Apps > Jupyter** (atau klik aplikasi Jupyter di halaman utama).
2.  Isi kolom **Number of CPU threads** sesuai kebutuhan alokasi core CPU Anda.
3.  Pilih opsi **Use GPU for the job**:
    *   Pilih **no** jika tanpa GPU.
    *   Pilih **yes** jika membutuhkan akselerasi GPU (akan dialokasikan 1x RTX 3090).
4.  Isi kolom **Number of hours** sebagai reservasi waktu maksimal sesi Jupyter, yang ditentukan oleh pilihan jumlah CPU, GPU, dan sisa Core Hour yang tersedia.
5.  Pada bagian **Choose package to run**, pilih **User provided**.
6.  Pada bagian **Choose how to launch Jupyter**, pilih **Anaconda / Mamba**.
7.  Pada bagian **Choose Anaconda/Mamba version**, pilih versi modul Lmod conda yang sesuai dengan lingkungan Anda.
8.  Pada bagian **User provided Conda/Mamba environment**, masukkan nama conda env user yang telah dibuat.
    *   Untuk memastikan nama modul pyload yang benar, Anda dapat membuka menu **Clusters > Aleleon Shell Access** lalu jalankan perintah `$ pyl info [nama-modul]` untuk melihat informasi modul tersebut.
9.  Klik tombol **[Launch]** untuk memulai sesi. User akan diarahkan ke halaman **My Interactive Sessions** dan harus menunggu hingga status berubah menjadi **Running**.

Setelah status menjadi **Running**, klik tombol **[Connect to Jupyter]** untuk mengakses sesi. Sistem mulai menghitung Core Hour saat sesi terhubung, dan sesi akan terus berjalan selama waktu sesi masih ada meskipun koneksi terputus atau user keluar dari web.
