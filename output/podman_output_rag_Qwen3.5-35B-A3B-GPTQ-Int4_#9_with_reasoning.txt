Berikut adalah rincian berdasarkan sumber yang tersedia:
1. Pada tabel "LFU - Setiap Job Komputasi" di bagian Limitasi Fair Usage, disebutkan bahwa limit maksimal waktu job untuk Akun Perseorangan adalah **3 hari (72 jam)**.
2. Untuk akun uji coba (Trial), batasnya lebih kecil yaitu **1 hari (24 jam)**, namun ini berbeda dengan akun perseorangan reguler.

Berikut adalah perbedaan dan spesifikasi antara kedua partisi berdasarkan dokumen referensi:

1. Partisi **torti**
   - Berjalan di interactive node CPU.
   - Tidak memiliki GPU.
   - Spesifikasi per node: AMD Threadripper 3990X, 32 v-core, 2.9Ghz, RAM 55GB efektif.
   - Digunakan jika Anda hanya membutuhkan komputasi CPU tanpa akselerasi GPU.

[Q22/69] Saya punya file simulasi.ipynb. Bagaimana urutan langkah menjalankannya sebagai batch job di Job Composer EWS menggunakan conda environment saya sendiri?
------------------------------------------------------------
Berikut adalah urutan langkah untuk menjalankan file simulasi.ipynb Anda sebagai batch job menggunakan template Conda Environment User di Job Composer EWS:

1. Login ke [ EFIRO Web Service ], buka menu Jobs > Job Composer, lalu buat ruang job dengan memilih menu New Jobs > From Template.
2. Pilih template "Conda Environment User (CPU)" jika berjalan di CPU atau "Conda Environment User (GPU)" jika membutuhkan akselerasi GPU, isi kolom Job Name (bebas), lalu klik tombol [Create New Job].
3. Siapkan file komputasi via menu [Edit Files] dengan batas upload 100MB per file; jika lebih besar, gunakan [ Aplikasi SFTP ] ke direktori Job Composer.
4. Karena Batch job hanya menjalankan file Python .py, lakukan konversi file Notebook Anda ke format Python dengan membuka [Open Terminal] dan jalankan perintah `jupytext --to py simulasi.ipynb` yang akan menghasilkan file simulasi.py.
5. Lakukan pengecekan kelengkapan environment dengan membuka [Open Terminal], mengaktifkan modul pyload user menggunakan perintah `pyl load nama-modul`, lalu jalankan `pycheck simulasi.py`. Jika output menampilkan "MISSING PACKAGE!", instal package yang kurang sesuai panduan (contoh: `$ pip install --user pyusb`).
6. Isi kolom submit script dengan klik [Open Editor], ikuti petunjuk NOTES di dalamnya, dan klik [Save] untuk menyimpan script.
7. Jalankan job dengan mengklik tombol [Submit].
8. Pantau status dan output job dengan merfresh halaman Job Composer; hasil output terminal akan tersedia di kolom Folder Contents dengan nama file `result-conda-jobID.txt` dan log error di `error-conda-jobID.txt`.

[Q32/69] (Troubleshooting) Saya submit job GROMACS tapi selalu gagal dengan pesan error Invalid syntax. Di script saya menulis #SBATCH --mem= 64 GB. Apa yang salah dari tulisan saya?
------------------------------------------------------------
Masalah pada penulisan perintah `#SBATCH --mem= 64 GB` terletak pada adanya spasi di antara angka dan satuan, serta penggunaan kata "GB" yang tidak sesuai dengan format dokumen.

Berdasarkan template submit script GROMACS versi CPU maupun GPU yang tersedia, spesifikasi memori harus ditulis tanpa spasi setelah tanda sama dengan dan hanya menggunakan angka satuan GB tanpa singkatan teks.

Berikut adalah perbaikan yang benar sesuai dokumen:
1. Hapus spasi setelah tanda sama dengan (=) pada baris `--mem`.
2. Hapus teks "GB" karena unit sudah didefinisikan di komentar atau konteks variabel, namun dalam contoh kode dokumen tertulis `#SBATCH --mem=////GB` sebagai placeholder, yang berarti Anda harus mengisi angkanya langsung diikuti huruf GB tanpa spasi di tengah nilai jika mengikuti pola umum Slurm, namun perhatikan bahwa dalam template dokumen tertulis `#SBATCH --mem=////GB` yang menunjukkan format isian adalah angka lalu disambung GB.
3. Perintah yang benar seharusnya ditulis seperti ini: `#SBATCH --mem=64GB`.

Jika Anda melihat contoh template di dokumen:
- Template CPU: `#SBATCH --mem=////GB`
- Template GPU: `#SBATCH --mem=////GB`

Maka format input yang valid adalah angka langsung diikuti "GB" tanpa spasi, contohnya: `#SBATCH --mem=64GB`. Penulisan `64 GB` (dengan spasi) akan menyebabkan error syntax karena Slurm menganggap "GB" sebagai argumen terpisah atau parsing gagal.

[Q33/69] Saya submit 3 batch job berturut-turut. Job 1 pakai 64 CPU. Job 2 pakai 32 CPU. Kenapa saat saya submit Job 3 yang butuh 64 CPU, statusnya malah PENDING dengan tulisan QOSMaxCpuPerUserLimit, padahal node epyc masih banyak yang kosong?
------------------------------------------------------------
Status PENDING dengan kode **QOSMaxCpuPerUserLimit** pada Job 3 Anda terjadi karena alokasi CPU yang diminta melebihi batas limit atau sisa limit komputasi (Fair Usage Limit/LFU) milik user tersebut, bukan karena kekosongan node fisik.

Berdasarkan dokumen referensi, berikut adalah penjelasan dan langkah-langkah yang harus dilakukan:

[Q35/69] Saya mau menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array. Setiap simulasi butuh 4 core CPU dan 8GB RAM. Di script, apakah saya harus menulis --cpus-per-task=40 atau --cpus-per-task=4?
------------------------------------------------------------
Untuk menjalankan 10 simulasi FLACS-CFD sekaligus menggunakan fitur Slurm Array dengan kebutuhan 4 core CPU per simulasi, Anda harus menulis `--cpus-per-task=4` pada script submit, bukan 40.

Berdasarkan dokumen referensi untuk template **FLACS-CFD (Array)**, berikut adalah penjelasan dan langkah-langkah yang benar:

1.  **Pengaturan Core per Task**:
    Dalam bagian spesifikasi reservasi job (SBATCH) pada template array, parameter `--cpus-per-task` menentukan jumlah core thread CPU yang dialokasikan untuk **satu tugas (task)** atau satu simulasi dalam array tersebut. Karena setiap simulasi membutuhkan 4 core, maka nilainya adalah 4.
    
    Perintah yang tepat di dalam script adalah:
    ```bash
    #SBATCH --cpus-per-task=4
    ```

2.  **Pengaturan Array Job**:
    Untuk menjalankan 10 simulasi sekaligus, Anda mengatur parameter `--array` pada baris terpisah untuk mendefinisikan rentang ID array dari 1 hingga 10.
    
    Perintah yang tepat di dalam script adalah:
    ```bash
    #SBATCH --array=1-10
    ```

3.  **Perhitungan Total Resource**:
    Dokumen menjelaskan bahwa total sumber daya dihitung berdasarkan perkalian antara konfigurasi per task dengan jumlah array.
    *   **Total CPU** = `cpus-per-task` * `array` = 4 * 10 = 40 core thread CPU secara keseluruhan.
    *   **Total Memori** = `mem` * `array`. Jika Anda ingin setiap simulasi mendapat 8GB RAM, pastikan Anda mengisi `--mem=8GB`. Maka total memori yang dialokasikan sistem akan menjadi 80GB (8GB * 10).

4.  **Contoh Struktur Script Bagian SBATCH**:
    Berikut adalah potongan kode yang sesuai dengan template **SW-GXN [Section: 5B. Batch Job dengan Slurm di Terminal]** untuk kasus Anda:

    ```bash
    # > Partisi CPU jumbo
    #SBATCH --partition=epyc-jumbo
    
    # > Jumlah task yaitu 1
    #SBATCH --ntasks=1
    
    # > Jumlah job array, contoh 10 array: 1-10
    #SBATCH --array=1-10
    
    # > Jumlah core thread CPU, minimal 2
    # > Total CPU = cpus-per-task * array
    #SBATCH --cpus-per-task=4
    
    # > Jumlah memori RAM (satuan GB)
    # > Total memori = mem * array
    #SBATCH --mem=8GB
    
    # > Total reservasi waktu job
    # > Format HH:MM:SS atau D-HH:MM:SS
    # ex, 1 hari: 24:00:00 atau 1-00:00:00
    #SBATCH --time=////
    
    # > File output terminal, %j -> ID job
    #SBATCH --output=result-flacs-%j.txt
    
    # > File output log, status, dan error
    #SBATCH --error=error-flacs-%j.txt
    ```

Jadi, jawaban singkatnya adalah gunakan `--cpus-per-task=4` karena nilai tersebut mewakili kebutuhan per simulasi individu dalam array, sedangkan jumlah total simulasi diatur melalui `--array=1-10`.

[Q39/69] Kenapa saat saya meminta alokasi #SBATCH --ntasks=7, sistem Slurm ALELEON akan membulatkannya menjadi 8 dan saya ditagih biaya untuk 8 core?
------------------------------------------------------------
Sistem Slurm ALELEON membulatkan alokasi core CPU ganjil menjadi genap ke atas demi optimasi performa karena SMT (Simultaneous Multi-Threading) diaktifkan pada semua node, di mana 2 thread dalam 1 core fisik tidak boleh dipakai oleh 2 job berbeda.

Berdasarkan aturan yang berlaku:
1. Alokasi minimum core CPU adalah 2 dan harus genap.
2. Jika Anda meminta jumlah alokasi CPU ganjil seperti 7, sistem akan otomatis membundarkannya menjadi 8 agar sesuai dengan kebijakan cgroup di mana setiap 1 core fisik (yang terdiri dari 2 thread) dialokasikan eksklusif untuk 1 concurrent job.
3. Contoh pembulatan yang tercantum dalam dokumen menyatakan bahwa jumlah 7 akan dibulatkan menjadi 8, dan 23 menjadi 24.

Oleh karena itu, meskipun Anda meminta 7 core, sistem mengalokasikan 8 core, dan biaya Core Hour Usage dihitung berdasarkan total alokasi yang diberikan sistem tersebut (8 core).

    📚 Sumber (10 chunks):
    • Sausage dan Aturan Metrik Penggunaan ALELEON → C. Alokasi Core CPU Minimum 2 dan Genap  (https://wiki.efisonlt.com/wiki/Sausage_dan_Aturan_Metrik_Penggunaan_ALELEON)
      💡 Why: Sumber ini secara langsung menjelaskan aturan bahwa alokasi core CPU harus genap dan mengaktifkan SMT, yang menjadi dasar pembulatan alokasi ganjil menjadi genap.

[Q56/69] Bagaimana cara membuat conda environment di aleleon?
------------------------------------------------------------
Untuk membuat conda environment di ALELEON, pertama-tama Anda harus membuka terminal melalui menu **Apps > Aleleon Shell Access** atau login via SSH. Selanjutnya, aktifkan modul Anaconda yang sesuai dengan perintah `$ module load Anaconda3/2024.02-1` (atau versi lain seperti `Anaconda3/2025.06-1` untuk Python 3.13). Setelah modul aktif, buat lingkungan baru dengan menjalankan perintah `$ conda create -n [nama-env] python=[versi-python]`, misalnya `$ conda create -n tensor python=3.11`. Jika Anda memiliki file YAML resep instalasi, Anda juga dapat menggunakan perintah `$ conda create -f [nama-file-yml].yml`.

[Q57/69] bagaimana cara menjalankan jupyter dengan conda environment sendiri?
------------------------------------------------------------
Untuk menjalankan sesi Jupyter menggunakan Conda Environment sendiri di ALELEON, Anda dapat mengikuti langkah-langkah berikut berdasarkan dokumen referensi:

1. Login ke web EFIRO dan buka menu **Interactive Apps > Jupyter** untuk mengisi formulir permintaan sesi.
2. Pada bagian **Choose how to launch Jupyter**, pilih opsi **Anaconda / Mamba**.
3. Pada bagian **Choose Anaconda/Mamba version**, pilih versi yang sesuai dengan modul Lmod conda user Anda (contoh: **Anaconda 3 2024.02-1**).
4. Pada bagian **Conda/Mamba environment directory** atau **User provided Conda/Mamba environment**, isi dengan nama Conda Environment yang telah Anda buat (misalnya `tensor`), bukan direktori kosong.
5. Pastikan Anda telah mengaktifkan Conda Environment tersebut sebelumnya melalui terminal **Aleleon Shell Access** menggunakan perintah `$ source activate [nama-conda-env]` untuk memastikan instalasi package sudah lengkap sebelum memulai sesi.
6. Klik tombol **[Launch]** untuk memulai sesi, tunggu hingga status berubah menjadi **Running**, lalu klik **[Connect to Jupyter]**.

Selama sesi berjalan, sistem akan menghitung Core Hour dan sesi akan tetap aktif meskipun koneksi terputus atau Anda keluar dari layanan, selama waktu sesi belum habis.

   - **Job Composer EWS**: User login ke [EFIRO Web Service], membuka menu [Jobs > Job Composer], membuat ruang job dengan menu [New Jobs > From Template], lalu menyiapkan file komputasi melalui fitur [Edit Files] atau [Aplikasi SFTP] ke direktori Job Composer.
   - **Terminal Slurm**: User membuka terminal ALELEON melalui opsi [Login SSH] atau menu [Clusters > Aleleon Shell Access] di [EFIRO Web Service], kemudian upload/menyiapkan file komputasi di HOME menggunakan [Aplikasi SFTP] (dengan limit 100MB per file).

2. **Penulisan Submit Script**
   - **Job Composer EWS**: User mengisi kolom submit script secara langsung di antarmuka web dengan cara mengklik tombol [Open Editor], mengikuti petunjuk NOTES, lalu menyimpannya dengan klik [Save].
   - **Terminal Slurm**: User membuat file submit script (format .sh) secara manual di terminal Linux, menyalin template yang disediakan dalam dokumen, mengisi bagian yang ditandai `////`, dan menyimpannya sebagai file lokal sebelum dijalankan.

3. **Pelepasan/Running Job**
   - **Job Composer EWS**: User menjalankan job dengan mengklik tombol [Submit] pada antarmuka web.
   - **Terminal Slurm**: User menjalankan job dengan mengetik perintah `$ sbatch [nama-submit-script]` di terminal.

4. **Pemantauan Status dan Output**
   - **Job Composer EWS**: User memantau status dengan me-refresh halaman Job Composer untuk melihat status (Not Submitted, Queue, Running, Completed, Failed). Output dapat dilihat di kolom [Folder Contents] (misalnya `result-jobID.txt` dan `error-jobID.txt`) atau melalui menu [Monitoring > Grafana] untuk utilisasi CPU/Memori/GPU.
   - **Terminal Slurm**: Pemantauan dilakukan melalui perintah terminal seperti `squeue` (tidak disebutkan eksplisit di teks tapi implikasi umum Slurm), namun dokumen lebih menekankan pada pengecekan output log (`output.log`, `script.sh`) jika terjadi masalah pada sesi interaktif atau desktop, sedangkan untuk batch job terminal, user mengandalkan file output yang dihasilkan oleh script tersebut.

5. **Penggunaan Template**
   - **Job Composer EWS**: Tersedia pilihan template spesifik sesuai software (seperti FLACS-CFD, CP2K, ORCA, OpenMX, SIESTA) yang dapat dipilih dari daftar nama atau pencarian (contoh: search: `flacs`).
   - **Terminal Slurm**: User harus memilih dan menyesuaikan template submit script yang tersedia di dokumen (seperti template Quantum ESPRESSO, FLACS-CFD Single/Array) secara manual tanpa bantuan pemilihan template otomatis di UI.

[Q67/69] Bagaimana langkah lengkap membuat conda env baru dan modul pyload dari awal?
------------------------------------------------------------
Berikut adalah langkah lengkap untuk membuat conda environment baru dan modul pyload di ALELEON berdasarkan panduan resmi:

1.  Buka terminal ALELEON melalui opsi **Login SSH** atau masuk ke **[ EFIRO Web Service ]**, lalu pilih menu **Clusters > Aleleon Shell Access**.
2.  Aktifkan modul Lmod conda yang sesuai dengan kebutuhan Anda; tim admin menyarankan versi terbaru (contoh: `Anaconda3/2025.06-1`), namun hindari versi `Anaconda3/2024.06-1` karena adanya laporan bug. Gunakan perintah seperti `$ ml Anaconda3/2025.06-1`.
3.  Buat conda environment baru dengan format perintah berikut, pastikan mendefinisikan versi Python untuk isolasi site-packages:
    *   Untuk Anaconda: `$ conda create -n [nama-env] python=[versi-python]`
    *   Untuk Mamba: `$ mamba create -n [nama-env] python=[versi-python]`
    *   Contoh spesifik: `$ conda create -n skripsi python=3.12`
4.  Aktifkan conda environment yang baru dibuat menggunakan perintah `$ source activate [nama-env]`.
5.  Buat modul pyload untuk aktivasi lingkungan tersebut berikutnya dengan menjalankan perintah `(env) $ pyl mod`, kemudian ikuti alur script yang muncul dengan memilih menu no.1.

[Q68/69] Apa saja status job di squeue dan artinya masing-masing?
------------------------------------------------------------
Berikut adalah daftar status job (ST/STATE) yang ditampilkan pada perintah `squeue` beserta penjelasannya:

1. **Tidak ada di squeue** – Job bersangkutan telah selesai.
2. **PD (PENDING)** – Job tertahan atau mengantri dan belum berjalan; pengguna dapat melihat alasan dengan memeriksa kolom `NODELIST(REASON)`.
3. **R (RUN)** – Job sedang berjalan.
4. **CG (COMPLETING)** – Job telah selesai dan sedang dalam proses *clean-up*.
5. **CA (CANCELED)** – Job dibatalkan oleh pengguna sendiri.
6. **PR (PREEMPETED)** – Job dibatalkan oleh tim admin, dengan notifikasi dan alasan dikirimkan melalui email support.
7. **S (SUSPENDED)** – Job ditahan oleh tim admin, dengan notifikasi dan alasan dikirimkan melalui email support.

[Q69/69] Bagaimana cara mengisi formulir Jupyter di EWS untuk conda env user?
------------------------------------------------------------
Berikut adalah panduan lengkap mengisi formulir Jupyter di EWS untuk Conda Environment User berdasarkan dokumen referensi:

1.  **Login ke Web EFIRO** dan buka menu **Interactive Apps > Jupyter** (atau klik aplikasi Jupyter di homepage).
2.  Pada bagian **Choose how to launch Jupyter**, pilih opsi **Anaconda / Mamba**.
3.  Pada bagian **Choose Anaconda/Mamba version**, pilih versi yang sesuai dengan modul pyload user Anda (biasanya **Anaconda 3 2024.02-1** atau versi terbaru yang tersedia).
4.  Pada bagian **User provided Conda/Mamba environment**, isi dengan nama conda environment yang telah dibuat, atau gunakan nama modul pyload jika merujuk pada info dari perintah `$ pyl info [nama-modul]`.
5.  Pada bagian **Partition**, pilih partisi sesuai kebutuhan komputasi:
    *   Pilih **torti** untuk **interactive node CPU**.
    *   Pilih **tilla** untuk **interactive node GPU**.
6.  Isi parameter alokasi sumber daya berikut sesuai batasan akun Anda:
    *   **Number of CPU thread(s)**: Masukkan jumlah core CPU dalam rentang **2 - 16** (untuk akun uji coba maksimal 16 core).
    *   **Amount of memory/RAM (GB)**: Masukkan jumlah RAM dalam satuan GB dalam rentang **1 - 55** (untuk akun uji coba maksimal 64GB).
    *   **Number of GPUs**: Jika memilih partition **tilla**, isi angka **1** (maksimal 1 unit untuk akun uji coba).
    *   **Number of hours**: Tentukan waktu maksimal sesi dalam rentang **1 - 24** jam (maksimal 24 jam untuk akun uji coba).
7.  Pada bagian **Software modules**, masukkan nama modul software tambahan jika diperlukan (misalnya modul CUDA jika tidak ada di dalam conda env 'tensor').
8.  Pada bagian **Conda/Mamba environment directory**, isi dengan nama conda env yang digunakan (contoh: **tensor**), kecuali jika menggunakan env lain maka isi dengan nama env tersebut.
9.  Isi kolom **Email address** jika ingin menerima notifikasi status mulai dan selesai sesi.
10. Klik tombol **Launch** untuk memulai sesi.

Setelah sesi berjalan, pastikan menggunakan kernel **Python 3 (ipykernel)** untuk menjalankan notebook.
